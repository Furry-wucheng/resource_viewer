# 阶段 01 — 项目脚手架与基础 UI

> **定位**：Foundation | **子阶段数**：6 | **前置依赖**：无
> **目标**：搭建可运行的 Flutter Desktop 应用骨架，包含主题系统、响应式导航框架和 3 个 Tab 空壳页

---

## 阶段总体目标

交付一个包含完整导航框架、主题系统和 Provider DI 注册的空白 Flutter 桌面应用。三个底部 Tab（首页/数据源/设置）可切换且保持状态，路由系统就绪。

## 1.2 主题与设计令牌

### 执行目标
将 `@design/design-tokens.css` 中的设计令牌翻译为 Flutter `ThemeData`，支持浅色/深色主题切换。

### 任务边界
- 不涉及任何页面 UI 组件，仅定义 Theme
- 颜色、间距、字体、圆角、阴影需完整覆盖

### 产出物
| 文件 | 说明 |
|------|------|
| `lib/ui/core/theme/app_theme.dart` | ThemeData 定义（浅色 + 深色） |
| `lib/ui/core/theme/app_colors.dart` | 颜色常量（12 色标签预设、功能色、中性色） |
| `lib/ui/core/theme/app_spacing.dart` | 间距常量（4/8/12/16/24/32） |
| `lib/ui/core/theme/app_text_styles.dart` | 文字样式（标题/正文/标签/文件名等） |

### 验收标准
- [ ] 浅色主题 `ThemeData` 含完整 colorScheme/textTheme/componentTheme
- [ ] 深色主题背景色与 `@design/design-tokens.css` 一致
- [ ] 12 色标签预设颜色值与 PRD `@prd/02-标签系统.md` §4.2 一致
- [ ] 查看器背景色强制黑色（不受主题切换影响）

## 1.5 响应式导航框架

### 执行目标
实现根据屏幕宽度自适应切换 BottomNavigationBar（< 900dp）和 NavigationRail（>= 900dp）的导航框架。

### 任务边界
- 仅导航框架本身，Tab 内容为占位 Scaffold
- 不涉及 Tab 内页面内容

### 产出物
| 文件 | 说明 |
|------|------|
| `lib/ui/core/widgets/app_shell.dart` | `MainShell` Widget，自适应 BottomNavigationBar / NavigationRail |

### 验收标准
- [ ] 宽度 < 900dp：显示 BottomNavigationBar，3 Tab（🏠 首页库 / 📂 数据源 / ⚙ 设置）
- [ ] 宽度 >= 900dp：显示 NavigationRail（常驻侧栏），3 Tab 图标 + 文字
- [ ] 窗口 resize 时导航组件实时切换
- [ ] Tab 切换时 `indexedStack` 保持各 Tab 页面状态
- [ ] 选中 Tab 高亮符合 Material Design 规范
- [ ] 与 `@design/README.md` 中描述的导航行为一致
