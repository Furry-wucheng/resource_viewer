# 阶段 04 — 标签系统与筛选

> **定位**：Core | **子阶段数**：10 | **前置依赖**：01（脚手架）, 02（数据层）
> **目标**：完整的标签 CRUD + 内置"收藏"标签 + 首页筛选栏交集筛选 + 标签管理页

---

## 阶段总体目标

交付完整的标签系统：标签创建/重命名/删除/颜色修改、资源-标签关联、首页横向筛选栏（全部/收藏/自定义标签交集筛选）、快捷收藏、批量打标签、标签管理页。此阶段完成后，用户可通过标签体系组织和检索资源。

## 4.1 标签 Repository（TagRepository）

### 执行目标
实现 Tag 和 ResourceTag 的全部 CRUD 操作及交集筛选查询。

### 任务边界
- 所有方法返回 `Result<T>`
- 交集查询使用 `GROUP BY HAVING COUNT(*) = N`

### 产出物
| 文件 | 说明 |
|------|------|
| `lib/data/repositories/tag_repository.dart` | `TagRepository` |

### 验收标准
- [ ] `createTag(name, color)` — 创建自定义标签，唯一性校验
- [ ] `renameTag(id, newName)` — 内置标签不可重命名
- [ ] `deleteTag(id)` — 内置标签不可删除，删除后级联清除 ResourceTag
- [ ] `updateColor(id, color)` — 修改标签颜色
- [ ] `getAllTags()` — 返回全部标签（内置在前，自定义按创建时间倒序）
- [ ] `getResourceTags(resourceId)` — 获取某资源的所有标签
- [ ] `addResourceTag(resourceId, tagId)` — 关联资源-标签
- [ ] `removeResourceTag(resourceId, tagId)` — 解除关联
- [ ] `setResourceTags(resourceId, tagIds)` — 全量替换某资源的标签（先删后插事务）
- [ ] `filterByTags(tagIds)` — 交集筛选（GROUP BY HAVING COUNT）
- [ ] `tagResourceCounts()` — 每个标签关联资源数统计
- [ ] `countFiltered(tagIds)` — 筛选结果计数（用于"筛选出 12/56"）
- [ ] 单元测试：交集筛选正确性（0/1/多标签）、级联删除、唯一约束

### 任务边界

### 产出物
| 文件 | 说明 |
|------|------|
| `lib/ui/features/tags/view_models/tag_view_model.dart` | `TagViewModel` |

### 验收标准
- [ ] `validateTagName(name, existingNames)` — 空/全空格/与内置同名/重复/超长
- [ ] 校验失败 → ValidationError → SnackBar/输入框红字
- [ ] `loadTags()` → loading → success/error
- [ ] `createTag(name, color)` → 成功后刷新列表
- [ ] `renameTag(id, newName)` → 更新后刷新
- [ ] `deleteTag(id)` → 二次确认 → 删除后刷新
- [ ] 单元测试：状态转换（loading→success、loading→error→retry）、校验规则
