# 阶段 06 — 组织模式与高级查看

> **定位**：Core | **前置依赖**：01~04（6.1~6.12 不依赖 05 SMB；6.13~6.15 依赖 05）
> **目标**：三种组织模式策略、自动判定、PDF 查看器、画廊模式、ResourcePicker、拆分资源

---

## 阶段总体目标

交付资源组织结构的完整实现：章节模式、平铺网格、画廊模式三种策略 + 自动判定 + 手动切换。原"直接阅读"场景统一归入平铺网格（单层文件自然退化为全文件网格 + 叶子文件夹自动进入阅读设置）。同时实现 PDF 查看器（与图片查看器统一体验）、ResourcePicker 弹窗组件、资源拆分功能、资源详情弹窗。

## 6.2 ChapterStrategy 实现

### 执行目标
实现章节模式：子文件夹作为章节，每个章节作为独立阅读单元。顶层散落文件显示在章节列表末尾。

### 任务边界
- 章节排序：按子文件夹名称自然排序
- 章节封面：**遍历子文件夹内文件列表**，找到第一张可用图片作为封面路径
- `getContents()` 返回子文件夹 + 顶层散落兼容文件

### 产出物
| 文件 | 说明 |
|------|------|
| `lib/shared/organization/chapter_strategy.dart` | `ChapterStrategy` |

### 验收标准
- [ ] `mode` = `OrganizationMode.chapter`
- [ ] `getChapters()` → 遍历子文件夹，返回 `List<Chapter>`（名称/路径/封面路径/页数）
- [ ] 封面路径：遍历子文件夹文件列表，取第一张图片（支持 jpg/jpeg/png/gif/webp/bmp）的路径
- [ ] 空子文件夹标记为 `isDisabled = true`（不可点击，列表置灰）
- [ ] `createProvider(chapter: chapter)` → 返回该章节的 `ImageFolderProvider`
- [ ] `getContents()` → 子文件夹列表 + 顶层散落兼容文件列表（章节在前，文件在后）

## 6.5 FlatGridStrategy 实现 + FlatGridPage

### 执行目标
实现平铺网格模式：当前层级网格展示，文件夹逐层进入（仅返回键），文件点击进入查看器。当资源仅含单层文件（无子文件夹）时，平铺网格退化为全文件缩略图网格（原"直接阅读"场景），并可通过全局设置跳过网格直接进入阅读。

### 任务边界
- **直接复用** `FileGridView` 和 `FileListView`（`lib/ui/features/sources/widgets/file_grid_view.dart` / `file_list_view.dart`），所有 source 专属参数（importedPaths / resourceTags / selectedEntries 等）不传
- FlatGridPage 是简化版 FileBrowserPage：无侧栏、无多选、无导入、无面包屑
- 下钻导航仅靠返回键（无面包屑），ViewModel 维护导航栈
- 支持 grid/list 视图切换，持久化到 SharedPreferences（复用 `file_browser_view_mode` 或独立 key）
- 全局设置 `autoEnterLeafFolder`（SharedPreferences，默认 false）：当资源仅含单层文件时，跳过网格直接进入查看器第一页翻页
- 不再单独实现 DirectReadStrategy 或 DirectReadGridPage——此场景完全由 FlatGrid 覆盖

### 产出物
| 文件 | 说明 |
|------|------|
| `lib/shared/organization/flat_grid_strategy.dart` | `FlatGridStrategy` |
| `lib/ui/features/viewer/flat_grid_page.dart` | `FlatGridPage`（包装 FileGridView + FileListView + 下钻） |
| `lib/ui/features/viewer/view_models/flat_grid_view_model.dart` | `FlatGridViewModel`（导航栈 + listDirectory） |

### 验收标准
- [ ] `mode` = `OrganizationMode.flatgrid`
- [ ] `getChapters()` → 空列表
- [ ] `getContents()` → 当前层级文件夹 + 兼容文件列表（文件夹优先 + 名称排序）
- [ ] FlatGridPage 顶栏：返回 + 资源名 + grid/list 切换按钮
- [ ] `FileGridView` 复用：`onTap` 文件夹 → push 新 FlatGridPage(subPath)；文件 → ViewerPage
- [ ] `FileListView` 复用：list 视图时使用，同切换逻辑
- [ ] 文件夹下钻仅靠返回键导航（无面包屑），ViewModel 维护 `_navigationStack`
- [ ] grid/list 视图切换持久化到 SharedPreferences
- [ ] 与 `@design/flat-grid.html` 布局一致

## 6.6 GalleryStrategy 实现（画廊模式）

### 执行目标
实现画廊模式：递归展开所有子文件夹的全部兼容文件到一个大网格。点击文件进入统一查看器，翻页范围覆盖全画廊扁平列表。

### 任务边界
- 递归遍历，不保留文件夹层级
- GalleryPage **复用** `FileGridView` 和 `FileListView`（与 FlatGridPage 共享同一套网格渲染组件，仅数据源不同：递归扁平 vs 当前层级）
- 翻页方案：使用 `FileSequenceViewerPage` 模式，构造全画廊 `List<ViewerMediaItem>` → `ViewerPage.media()`
- 大量文件（>1000）暂不做虚拟滚动优化（P2 处理）

### 产出物
| 文件 | 说明 |
|------|------|
| `lib/shared/organization/gallery_strategy.dart` | `GalleryStrategy` |
| `lib/ui/features/viewer/gallery_page.dart` | `GalleryPage`（复用 FlatGridPage 同款 FileGridView 布局 + 数据源差异） |
| `lib/ui/features/viewer/view_models/gallery_view_model.dart` | `GalleryViewModel`（递归扁平化 FileEntry 列表） |

### 验收标准
- [ ] `mode` = `OrganizationMode.gallery`
- [ ] `getContents()` → 递归扁平化全部兼容文件列表
- [ ] 排序：默认按文件夹名 → 文件名字母升序
- [ ] `getChapters()` → 空列表
- [ ] GalleryPage 顶栏：返回 + 资源名 + "画廊模式" 标签 + 总文件数
- [ ] GalleryPage 使用 FlatGridPage 同款布局（复用 `FileGridView`），仅数据源不同（递归扁平 vs 当前层级）
- [ ] 点击文件 → 构造全画廊 `List<ViewerMediaItem>`（复用 `FileSequenceViewerPage` 模式），翻页范围覆盖整个画廊列表
- [ ] `ViewerPage.media()` 翻页范围覆盖画廊全部文件（图片+视频混合）
- [ ] 与 `@design/flat-grid.html` 布局一致（画廊模式仅数据源为递归扁平，网格渲染相同）
