# 阶段 06 — 组织模式与高级查看

> **定位**：Core | **子阶段数**：14 | **前置依赖**：01~05
> **目标**：四种组织模式策略、自动判定、PDF 查看器、画廊模式、ResourcePicker、拆分资源

---

## 阶段总体目标

交付资源组织结构的完整实现：直接阅读、章节模式、平铺网格、画廊模式四种策略 + 自动判定 + 手动切换。同时实现 PDF 查看器（与图片查看器统一体验）、ResourcePicker 弹窗组件、资源拆分功能、资源详情弹窗。

## 6.8 自动模式判定（DetectOrganizationModeUseCase）

### 执行目标
实现文件夹结构的自动分析，推荐最优组织模式。

### 任务边界
- 判定逻辑按 `@prd/07-资源组织结构.md` §3.1：
  - 含图片无子文件夹 → direct
  - 含子文件夹且子文件夹内含图片 → chapter
  - 混合内容（图片 + PDF + 子文件夹） → flatgrid
  - 无法判定 → flatgrid（兜底）

### 产出物
| 文件 | 说明 |
|------|------|
| `lib/domain/use_cases/detect_organization_mode_use_case.dart` | `DetectOrganizationModeUseCase` |

### 验收标准
- [ ] 扫描入库 / 拆分资源后自动调用
- [ ] 结果写入 `Resource.organizationMode`
- [ ] 判定为 null 时（待判定）不阻塞正常使用
- [ ] 用户可手动修改（6.9）
- [ ] 单元测试：direct/chapter/flatgrid 各场景的判定准确性

## 6.14 ResourcePicker 弹窗组件

### 执行目标
实现可复用的树形扫描选择弹窗：纯树形结构、节点勾选（不级联）、全选子项、智能识别规则。

### 任务边界
- `showDialog` 弹窗，不走路由
- 三个调用入口：文件浏览器"扫描入库"、批量添加、首页"拆分资源"

### 产出物
| 文件 | 说明 |
|------|------|
| `lib/ui/features/sources/widgets/resource_picker_dialog.dart` | `ResourcePickerDialog` |

### 验收标准
- [ ] 顶部栏："关闭 | 标题 | 已选 N 项"
- [ ] 纯树形结构，无右侧内容区
- [ ] **勾选规则**：勾选 = 该节点为一个独立资源，不级联不递归
- [ ] 点击节点名/展开箭头 → 展开/折叠子树（纯导航，不改变勾选）
- [ ] **全选子项**按钮：仅当有子文件夹时显示，一键勾选直接子项（仅一层）
- [ ] 复选框出现条件：仅显示在可作资源单元的节点上
- [ ] 智能识别：文件夹下一层全是图片 → 叶子节点不可展开；含子文件夹 → 可展开
- [ ] 空文件夹不显示
- [ ] 文件类型过滤：仅展示文件夹 + 兼容文件（图片/PDF/压缩包/视频）
- [ ] 底部按钮随场景变化：批量添加 → `[批量添加资源] [批量打标签]`；拆分 → `[拆分并保留原] [拆分并删除原]`
- [ ] 宽屏/窄屏同一布局，仅弹窗尺寸不同
- [ ] 与 `@design/resource-picker.html` 交互一致
- [ ] Widget 测试：勾选逻辑、全选子项仅一层、已选计数
