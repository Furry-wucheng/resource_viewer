## 代码审查发现

* [x] **目录封面没有递归查找第一张可用图片**  
  已修复：资源库封面生成 `ImageThumbnailGenerator.generate` 继续使用较深的受限递归；源文件浏览即时预览 `ThumbnailRepository._previewDirectory` 不递归也不进入子目录，只扫描当前文件夹直属文件，避免浏览时深扫目录。

* [x] **进入目录时存在 N+1 标签查询，资源多时会卡顿**  
  已修复：新增 `AppDatabase.getTagsForResources` / `TagRepository.getTagsForResources`，`FileBrowserViewModel` 进入目录后批量加载标签，避免逐资源串行查询。

* [x] **缩略图生成可增加小图直通路径**  
  已修复：JPEG/PNG 小图在尺寸和文件体积均低于阈值时直接复用原始字节，避免不必要的缩放与重编码。

* [x] **“智能判定”文案需明确为延迟判定**  
  已修复：批量添加弹窗文案调整为“打开时自动判定”，对应 `organizationMode = null` 后在资源查看器中延迟执行 `DetectOrganizationModeUseCase` 的现有行为。

* [ ] **新增交互缺少 Widget/ViewModel 覆盖**  
  当前仅调整了文件浏览器 ViewModel 测试。首页多选批量删除、批量添加 Dialog 的标签/组织模式返回值、资源详情 Dialog 布局还没有对应 Widget 或 ViewModel 测试，后续改动容易回归。
