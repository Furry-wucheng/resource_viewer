## 使用说明

1. 每个阶段文件包含该阶段的**总体目标**和**前置依赖**
2. 每个子阶段包含：执行目标 / 任务边界 / 产出物 / 验收标准
3. 验收标准中的 `@design/xxx.html` 引用指向 `doc/design/` 下的交互原型
4. 验收标准中的 `@prd/` 引用指向 `doc/prd/` 下的需求文档
5. 验收标准中的 `@tech/` 引用指向 `doc/tech/` 下的技术文档
6. 各阶段**顺序执行**，但阶段 03 和 04 可**并行开发**（仅共享数据层依赖）

## 约束

- 每个子阶段的验收标准必须**可验证**（自动化测试或手动检查清单）
- 每个子阶段完成后需通过 `flutter analyze` 零警告
- 每个子阶段对应的测试文件需在该子阶段内完成
- ViewModel 不使用 try/catch（遵循 `@tech/05-错误处理策略.md`）
- Repository 所有返回值包装为 `Result<T>`（遵循 `@tech/05-错误处理策略.md`）
