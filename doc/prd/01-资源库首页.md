# 01 — 资源库首页

> 所属模块：A · 优先级：P0 · 版本：v1.2 · 日期：2026-06-17 · 状态：已定稿

---

## 1. 模块概述

资源库首页是用户打开应用后见到的第一个页面。它以一个**自适应竖版缩略图网格**将所有已添加的资源平铺展示，每项资源显示一张竖版封面缩略图和其名称。用户可以通过顶部筛选栏切换视图：全部资源、收藏资源、或按自定义标签交集筛选。

资源通常是添加进去的**文件夹**（少数为 PDF 或压缩包文件），视频一般也内嵌在文件夹中。因此首页只需对 PDF 和压缩包显示类型角标。

> 资源内部的组织方式（章节模式 / 平铺网格 / 画廊模式）详见 [@07-资源组织结构.md](07-资源组织结构.md)。

## 2. 用户故事

| ID | 角色 | 场景 | 期望结果 |
|----|------|------|----------|
| US-A01 | 收藏用户 | 打开应用，想看看最近收藏了哪些漫画 | 首页以网格展示所有资源缩略图，一目了然 |
| US-A02 | 收藏用户 | 漫画以文件夹形式存放 | 首页以该文件夹名作为资源名，显示封面缩略图 |
| US-A03 | 收藏用户 | 资源数量多（100+），想快速浏览并找到某个漫画 | 页面滚动流畅，有排序和筛选手段 |
| US-A04 | 收藏用户 | 想区分哪些是 PDF 或压缩包（与普通文件夹区分） | PDF 和压缩包在缩略图右下角显示对应角标 |
| US-A05 | 收藏用户 | 首次使用还没添加任何资源 | 看到友好的空状态引导，提示去添加数据源 |
| US-A06 | 收藏用户 | 某个资源缩略图加载失败 | 显示该资源类型的通用占位图，不影响整体浏览 |
| US-A07 | 收藏用户 | 想把几个常看的漫画标记为收藏，方便快速找到 | 将资源标星收藏，点击"收藏"筛选项即可只看收藏的资源 |
| US-A08 | 收藏用户 | 导入资源时误以父文件夹为粒度，想把里面的子文件夹拆出来 | 长按资源 → 拆分资源 → 勾选子文件夹 → 拆成独立资源 |

## 4. 详细需求

### 4.1 缩略图网格

#### 4.1.1 列数自适应

| 屏幕宽度 | 网格列数 |
|----------|----------|
| < 600dp（手机竖屏） | 3 列 |
| 600 ~ 900dp（手机横屏 / 小平板） | 4 列 |
| 900 ~ 1200dp（大平板） | 5 列 |
| > 1200dp（桌面） | 6 列 |

#### 4.1.2 单项布局

缩略图为**竖长方形**（宽高比约 2:3 或 3:4），图片居中裁切（cover 模式），不做卡片包裹，文件名直接显示在缩略图下方。

```
 ┌──────┐
 │      │
 │ 缩   │
 │ 略   │  ← 竖长方形，比例约 2:3 ~ 3:4
 │ 图   │
 │      │
 └──────┘
 文件名……
```

- 缩略图宽高比：竖长方形（建议约 2:3）
- 文件名：12sp，居中，最多 2 行，溢出省略号
- 无卡片容器，无背景框，文件名和缩略图之间 4dp 间距
- 项间距：8dp
- 内边距：12dp

#### 4.1.3 交互

| 操作 | 行为 |
|------|------|
| 单击 | 进入资源查看器 |
| 长按（移动端）/ 右键（桌面端） | 弹出菜单：打开 / **拆分资源** / **打标签** / **属性** / **删除** / 进入批量选择 |
| 批量选择模式下单击 | 切换该项选中/取消选中 |
| 单击缩略图左上角收藏图标（或查看器内操作） | 切换收藏/取消收藏 |

### 4.2 缩略图生成规则

| 资源类型 | 缩略图来源 | 优先级 |
|----------|-----------|--------|
| 图片文件夹 | 取文件夹内第一张图片（按文件名排序） | P0 |
| PDF 文件 | 渲染第一页为缩略图 | P0 |
| 压缩包（zip/rar/7z） | 取包内第一张图片（不解压） | P1 |
| 视频文件 | 取视频首帧或视频占位图标 | P0 |
| 未知 / 无可提取内容 | 显示类型通用占位图 | P0 |

**缓存策略**：缩略图生成后缓存到本地缩略图目录，后续直接从缓存加载。资源内容变更时重新生成。

**占位图与类型角标**（PDF、压缩包和视频需要）：

| 类型 | 占位图标 | 说明 |
|------|---------|------|
| PDF | PDF 文件图标 | 缩略图右下角角标 |
| 压缩包 | 压缩包图标 | 缩略图右下角角标 |
| 视频 | 视频文件图标 | 缩略图右下角角标 |
| 文件夹（默认） | 无角标 | 大部分资源为文件夹，不需要标记 |
| 通用未知 | 问号图标 | 无法识别类型时使用 |

### 4.3 筛选栏

#### 4.3.1 布局

筛选栏位于首页顶部，资源网格上方。固定顺序如下：

```
┌──────────────────────────────────────────────────┐
│ [🔍 搜索框]                        [⚙ 管理]     │  ← 搜索框 + 管理按钮
│ [🔲 全部] [⭐ 收藏] [热血] [已完结] [汉化] ...  │  ← 横向滚动
└──────────────────────────────────────────────────┘
 |            |          └─────────────────────┘
 |            |               自定义标签（可滚动）
 |            └─ 系统内置：收藏（不可删除）
 └─ 固定：全部
                                            └─ 管理按钮：跳转标签管理页
```

- 顶部搜索框：按资源名称模糊匹配，与标签筛选取交集（无标签筛选时搜全部）
- "管理"按钮位于筛选栏右侧，点击跳转标签管理页（见 `@02-标签系统.md` §4.6）
- "全部"和"收藏"固定在左侧，不随标签列表滚动
- "全部"和"收藏"样式区别于自定义标签（例如实心图标 vs 颜色圆点）
- 自定义标签支持横向滚动，超出屏幕时可滑动

#### 4.3.2 交互规则

| 操作 | 行为 |
|------|------|
| 在搜索框输入 | 按资源名称模糊匹配，与当前标签筛选取交集 |
| 点击"全部" | 取消所有筛选（包括收藏和搜索框清空），显示全部资源 |
| 点击"收藏" | 仅显示被收藏的资源（拥有内置"收藏"标签） |
| 未选中任何标签 | 显示全部资源（默认状态） |
| 点击未选中自定义标签 | 选中该标签，列表仅显示拥有此标签的资源 |
| 点击已选中自定义标签 | 取消选中，列表更新 |
| 同时选中收藏 + 自定义标签 | 取交集：收藏的 且 有该标签的资源 |
| 选中多个自定义标签 | 取**交集**（同时拥有所有选中标签的资源） |

**视觉反馈**：
- "全部"默认选中高亮；点击其他筛选项时"全部"自动取消高亮
- 选中的标签/筛选项高亮显示（"收藏"用星标高亮，自定义标签用颜色背景）
- 未选中的灰色描边样式

**筛选结果提示**：
- 筛选后显示提示文本："筛选出 12 / 56 个资源"
- 筛选结果为空时显示："没有匹配的资源，试试调整筛选条件"

### 4.4 状态管理

#### 4.4.1 空状态

当资源库中没有任何资源时：
- 显示一个大图标 + 提示文案："还没有资源"
- 下方显示操作引导按钮："去添加数据源"
- 点击跳转到数据源管理页面（Tab 1）

#### 4.4.2 加载状态

当数据源正在扫描中：
- 顶部显示扫描进度条或加载动画
- 已扫描出的资源正常显示
- 扫描完成的资源数量实时更新

#### 4.4.3 错误/降级状态

| 场景 | 处理 |
|------|------|
| 缩略图文件丢失 | 显示类型占位图 |
| 缩略图加载失败 | 显示类型占位图，不重试 |
| 源不可达（SMB 断网） | 该源资源置灰，显示"源不可用"标记 |

### 4.5 拆分资源

当用户误以父文件夹粒度添加为资源（如将 `/全部漫画/` 而非 `/全部漫画/海贼王/` 导入），可在首页将一个大资源拆散为多个独立子资源。

**入口**：长按/右键资源 → "拆分资源"

**流程**：
```
长按资源 → "拆分资源"
  → 打开 ResourcePicker 弹窗（纯树形扫描选择，见 @06-页面结构与交互.md）
  → 用户勾选子文件夹/文件
  → 底部两个按钮：
      [拆分并保留原资源] → 标签弹窗 → 创建子资源 + 原资源不变
      [拆分并删除原资源] → 二次确认："原资源'全部漫画'将被删除，拆出 N 个子资源。确定？"
                       → 标签弹窗 → 创建子资源 + 删除原资源 + 级联清除原资源标签关联
```

**约束**：
- 拆出的子资源基于 sourceId + relativePath 去重
- 空文件夹不在树中显示
- 拆分后子资源重新自动判定组织模式（见 `@07-资源组织结构.md`）
- 标签完全重新选择，不继承原资源标签
- ResourcePicker 弹窗的标签选择复用 `@02-标签系统.md` 组件

### 4.6 删除单个资源

当用户想移除某个资源但保留数据源时，可在首页删除单个资源（区别于删除整个数据源）。

**入口**：长按/右键资源 → "删除"

**流程**：
```
长按资源 → "删除"
  → 二次确认弹窗："确定删除资源'海贼王_卷01'？其标签关联将一并清除。"
  → 确认 → 删除 Resource 记录 + 级联删除 ResourceTag 关联
  → 缩略图缓存清理（可选，LRU 会自然淘汰）
  → 首页列表刷新，该资源消失
```

**约束**：
- 删除单个资源不影响同数据源的其他资源
- 删除单个资源不影响数据源本身（源仍保留，可重新扫描入库）
- 级联删除该资源的所有 ResourceTag 关联，标签本身保留
- 二次确认弹窗明确提示"标签关联将一并清除"
- 删除操作不可撤销（P0 不做回收站）

## 5. 边界条件

1. **资源数量极大（10000+）**：P1 虚拟滚动解决，P0 暂不做特殊处理
2. **文件夹内无图片**：不显示，扫描时自动跳过空资源
3. **文件名超长（>100 字符）**：超过 2 行截断，完整名称在详情弹窗中展示
4. **同名资源**：不做去重，两个资源独立展示（可能来自不同数据源）
5. **图片格式不支持**：跳过该图，继续尝试下一张；全部不可用则显示占位图
