# 05 — 数据模型

> 跨模块 · P0 · 状态：已定稿

---

## 1. 概述

本文档定义 Resource Viewer 中的核心数据实体、字段规格及实体间关系。所有模块的数据操作均基于此模型。

---

## 2. 实体定义

### 2.1 数据源（Source）

代表一个用户添加的文件来源。

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `id` | UUID | 是 | 唯一标识 | `a1b2c3d4-...` |
| `name` | String(100) | 是 | 用户自定义名称 | `"D盘漫画"` |
| `type` | Enum | 是 | 源类型 | `local` / `smb` / `ftp` / `webdav` |
| `rootPath` | String(500) | 是 | 根路径 | `"D:\\Comics"` 或 `"smb://192.168.1.100/share"` |
| `host` | String(255) | 否 | 主机地址（网络源） | `"192.168.1.100"` |
| `port` | Int | 否 | 端口号 | `445` |
| `username` | String(100) | 否 | 认证用户名 | `"admin"` |
| `password` | — | 否 | 认证密码 | 不入库，由 `flutter_secure_storage` 加密存储（见 `@tech/04-数据库设计.md` §4.1），Source 表仅存 `passwordStored` 标记 |
| `domain` | String(100) | 否 | 域/工作组 | `"WORKGROUP"` |
| `enabled` | Bool | 是 | 是否启用 | `true` |
| `isAvailable` | Bool | 是 | 当前是否可达 | `true` |
| `lastCheckAt` | DateTime | 否 | 上次连接检查时间 | `2026-06-17 10:30:00` |
| `createdAt` | DateTime | 是 | 创建时间 | |
| `updatedAt` | DateTime | 是 | 最后修改时间 | |

### 2.2 资源（Resource）

代表一个可展示/阅读的内容单元。

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `id` | UUID | 是 | 唯一标识 | |
| `sourceId` | UUID | 是 | 所属数据源 ID | |
| `name` | String(255) | 是 | 资源名称 | `"海贼王_卷01"` |
| `type` | Enum | 是 | 资源类型 | `folder` / `pdf` / `archive` |
| `organizationMode` | Enum | 否 | 组织模式 | `direct` / `chapter` / `flatgrid` / `gallery`（默认自动判定，见 `@07-资源组织结构.md`） |
| `relativePath` | String(1000) | 是 | 相对于源根目录的路径 | `"海贼王\\卷01_东海篇"` |
| `thumbnailPath` | String(500) | 否 | 本地缓存的缩略图路径 | `"/cache/thumb_abc123.jpg"` |
| `fileCount` | Int | 否 | 内部文件数（图片/页数） | `176` |
| `fileSize` | Int64 | 否 | 总文件大小（字节） | `536870912` |
| `isAvailable` | Bool | 是 | 文件当前是否可访问 | `true` |
| `lastScannedAt` | DateTime | 否 | 最后扫描时间 | |
| `createdAt` | DateTime | 是 | 创建时间 | |
| `updatedAt` | DateTime | 是 | 最后修改时间 | |

### 2.3 标签（Tag）

标签分为内置标签和自定义标签，由 `isBuiltIn` 区分。内置标签不可删除、不可重命名。

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `id` | UUID | 是 | 唯一标识 | |
| `name` | String(20) | 是 | 标签名称（唯一） | `"热血"` |
| `color` | String(7) | 是 | HEX 颜色值 | `"#E53935"` |
| `isBuiltIn` | Bool | 是 | 是否为内置标签（内置不可删除/重命名） | `false` |
| `createdAt` | DateTime | 是 | 创建时间 | |
| `updatedAt` | DateTime | 是 | 最后修改时间 | |

**内置标签数据**（首次安装时自动创建）：

| name | color | isBuiltIn |
|------|-------|-----------|
| 收藏 | `#FFC107` | `true` |

### 2.4 资源-标签关联（ResourceTag）

多对多关联表。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `resourceId` | UUID | 是 | 资源 ID |
| `tagId` | UUID | 是 | 标签 ID |
| `createdAt` | DateTime | 是 | 关联创建时间 |

**联合唯一约束**：`(resourceId, tagId)`

## 3. 实体关系图

```
┌──────────┐         ┌──────────────┐         ┌──────────┐
│  Source  │ 1 ─── N │   Resource   │ N ─── M │   Tag    │
│          │         │              │         │          │
│  id (PK) │         │  id (PK)     │         │  id (PK) │
│  name    │         │  sourceId(FK)│         │  name    │
│  type    │         │  name        │         │  color   │
│  path    │         │  type        │         │          │
│  ...     │         │  path        │         └──────────┘
└──────────┘         │  ...         │              │
                     └──────────────┘              │
                            │                      │
                            │     ┌────────────┐   │
                            └─────│ResourceTag │───┘
                                  │            │
                                  │ resourceId │
                                  │ tagId      │
                                  └────────────┘
```

**关系说明**：

| 关系 | 类型 | 说明 |
|------|------|------|
| Source → Resource | 一对多 | 一个数据源下有多个资源；删除源时级联删除其资源 |
| Resource → Tag | 多对多 | 通过 ResourceTag 关联；删除资源时级联删除关联 |
| Tag → Resource | 多对多 | 删除标签时级联删除关联，不影响资源 |

## 4. 数据流转

### 4.1 添加资源流程

```
用户添加数据源 → 文件浏览器勾选路径
  → 系统遍历被勾选项 → 生成 Resource 记录（填充 sourceId, name, type, relativePath）
  → 生成缩略图 → 写入 thumbnailPath
  → 资源出现在首页
```

### 4.2 标签关联流程

```
用户编辑资源标签
  → 展示全量 Tag 列表（勾选已关联的）
  → 用户勾选/取消
  → 对比变更 → 新增或删除 ResourceTag 记录
  → 首页筛选即时生效
```

### 4.3 筛选流程

```
用户点击标签 [A] → 查询：SELECT Resource WHERE id IN (
  SELECT resourceId FROM ResourceTag WHERE tagId = A
)
用户加选标签 [B] → 查询：SELECT Resource WHERE id IN (
  SELECT resourceId FROM ResourceTag WHERE tagId = A
  INTERSECT
  SELECT resourceId FROM ResourceTag WHERE tagId = B
)
→ 首页网格只显示筛选结果
```

### 4.4 同步流程

```
用户触发同步 → 遍历源的 rootPath → 对比已有 Resource 的 relativePath
  → 新增的文件 → 创建 Resource
  → 消失的文件 → Resource.isAvailable = false（而非直接删除，保留标签关联）
  → 修改的文件 → 更新 fileSize, fileCount, 重新生成缩略图
```

## 5. 数据生命周期

| 操作 | 对 Resource 的影响 | 对 Tag 的影响 | 对 ResourceTag 的影响 |
|------|-------------------|---------------|----------------------|
| 删除源（手动确认后） | 级联删除 | 不影响（Tag 保留） | 级联删除 |
| 删除资源 | 删除 | 不影响 | 级联删除 |
| 删除标签 | 不影响 | 删除 | 级联删除 |
| 资源文件不可达 | `isAvailable = false` | 不影响 | 不影响 |
| 资源文件恢复可达 | `isAvailable = true` | 不影响 | 不影响 |
