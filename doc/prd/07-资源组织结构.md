# 07 — 资源组织结构

> 所属模块：跨 A/D · 优先级：P0 · 状态：已定稿

---

## 1. 模块概述

一个资源（通常是一个文件夹）内部可能有不同的文件组织方式。用户在文件浏览器中将一个文件夹添加为资源后，系统需要知道该资源内部是"如何组织的"，以便在查看器中以正确的方式呈现内容。

本模块定义资源的三种内部组织模式，以及用户如何切换和管理它们。

---

## 2. 三种组织模式

### 模式 1：章节模式（Chapter Mode）

> 文件夹内包含多个子文件夹（章节），每个子文件夹视为一个阅读单元。

**适用场景**：按章节/卷整理的漫画。

**目录结构示例**：

```
/海贼王/
  ├── 第001话_冒险的黎明/
  │   ├── 01.jpg
  │   ├── 02.jpg
  │   └── ...
  ├── 第002话_那个男人/
  │   ├── 01.jpg
  │   └── ...
  └── ...
```

**查看器行为**：

- 进入后先展示章节列表（每个子文件夹作为一个章节项）
- 章节列表页采用**响应式布局**：宽屏（≥900dp）时左侧显示资源封面，右侧显示章节网格/列表；窄屏时封面在上，章节在下
- 支持 **grid/list 视图切换**（全局设置 `chapterViewMode`，默认 grid），切换按钮在页面顶部栏中
- 章节项显示：章节封面缩略图（遍历子文件夹找第一张可用图片）+ 章节名 + 页数
- 空章节置灰标记"空章节"，不可点击
- **顶层散落文件**：如果顶层有非文件夹的兼容文件（图片/PDF/视频等），显示在章节列表末尾（章节在前，文件在后），可点击进入查看器
- 点击章节 → 进入统一查看器翻阅该章节
- 章节之间支持跨章节连续阅读：末页继续左滑提示"下一章: xxx"，松手进入下一章首页（见 [@04-资源查看器.md](04-资源查看器.md) 4.1.4）

**章节排序**：按子文件夹名称排序（支持自然排序，即"第2话"排在"第10话"前面）

---

### 模式 2：平铺网格（Flat Grid）

> 以网格展示当前目录下的所有文件夹和兼容文件。与文件浏览器网格视图体验一致。当文件夹仅有一层图片文件（无子文件夹）时，平铺网格自然退化为全文件缩略图网格。

**适用场景**：混合内容目录、杂项收藏夹，以及单层图包/画集（原"直接阅读"场景）。

**目录结构示例**：

```
/杂项收藏/
  ├── cosplay合集.jpg
  ├── 设定集.pdf
  ├── 花絮.mp4
  ├── 画集/
  │   ├── 01.jpg
  │   └── 02.jpg
  └── ...
```

**查看器行为**：

- 进入后以网格展示当前层级的所有文件夹和兼容文件
- 文件夹显示封面缩略图 + 名称，文件显示自身缩略图 + 名称
- 点击文件 → 进入统一查看器，左右滑动浏览**当前平铺网格**中的兼容文件
- 点击文件夹 → 进入该子文件夹的平铺网格视图（逐层进入，非递归展开）
- 返回键 → 返回上一级目录的平铺网格
- **叶子文件夹自动进入**：当资源仅含一层图片文件（无子文件夹）时，可通过全局设置 `autoEnterLeafFolder` 跳过缩略图网格，直接进入查看器第一页翻页（见 [@08-设置页面.md](08-设置页面.md) §4.3）

---

### 模式 3：画廊模式（Gallery）

> **递归展开**所有子文件夹的全部兼容文件到一个大网格中，打破层级，一屏浏览所有内容。

**适用场景**：从各种渠道收集的散图、没有章节概念的无序图包。

**目录结构示例**：

```
/散图合集/
  ├── 子文件夹A/
  │     ├── 01.jpg
  │     └── 02.jpg
  ├── 子文件夹B/
  │     └── 03.jpg
  ├── standalone.jpg
  └── doc.pdf
```

**查看器行为**：

```
进入后平铺展开为：

┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│ 子文件夹A │ │ 子文件夹A │ │ 子文件夹B │ │standalone│ │  doc     │
│  01.jpg  │ │  02.jpg  │ │  03.jpg  │ │  .jpg   │ │  .pdf   │
└──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘
```

- **展开规则**：递归遍历所有子文件夹，收集全部兼容文件，平铺到一个网格
- **排序**：默认按文件夹名 → 文件名（字母升序）排列
- **点击文件** → 进入统一查看器，左右滑动浏览**画廊中所有文件的扁平列表**
  - 翻页方案：使用 `ViewerPage.media()` 模式，构造全画廊文件的 `ViewerMediaItem` 列表，翻页范围覆盖全部扁平文件
  - 文件类型混合处理：图片直接渲染，PDF 逐页展开为多个 ViewerMediaItem，视频作为独立项混入翻页流
- 回到画廊页面：顶栏显示资源名 + 总文件数
- 大量文件时（>1000）：P2 虚拟滚动

**画廊模式排序示例**：

```
子文件夹A/01.jpg
子文件夹A/02.jpg
子文件夹B/03.jpg
standalone.jpg
doc.pdf
```

**约束**：

- 画廊模式不保留文件夹层级信息——所有文件视为平面列表
- 子文件夹过多（>100 个）时，画廊模式可能加载缓慢，提示用户考虑使用平铺网格模式

## 3. 模式选择

### 3.1 默认行为（自动判定）

系统根据文件夹结构**自动推荐**模式，用户可手动修改。判定按以下顺序**从上到下**匹配，命中即停止（行间互斥）：

| 顺序 | 条件                                                    | 推荐模式 | 判定逻辑             |
| ---- | ------------------------------------------------------- | -------- | -------------------- |
| 1 | 文件夹内含 ≥1 个子文件夹，且**至少一个子文件夹内含图片**，且顶层**无混合文件类型**（顶层仅有子文件夹，无散落图片/PDF/视频） | 章节模式 | 存在明显章节/卷层级  |
| 2 | 文件夹内含混合文件类型（顶层有图片 + PDF + 视频等散落文件）+ 子文件夹，或仅有单层图片文件（无子文件夹） | 平铺网格 | 混合结构 或 单层结构，按层级导航 / 直接浏览 |
| 3 | 无法判定（空文件夹/无识别文件）                         | 平铺网格 | 兜底方案             |

**歧义消解规则**：
- 顶层有散落图片 + 子文件夹（子文件夹内有图片）→ 命中顺序 2（平铺网格），不命中顺序 1
- 顶层仅子文件夹（无散落图片），子文件夹内有图片 → 命中顺序 1（章节模式）
- 顶层仅有图片（无子文件夹）→ 命中顺序 2（平铺网格），即原"直接阅读"场景自动归入平铺网格
- 即"顶层是否有散落图片/文件"是区分章节模式与平铺网格的关键；"无子文件夹"的场景不再单独成模式

### 3.2 用户手动切换

**入口**：

- 资源详情弹窗 → "组织模式" 选项 → 选择三种之一
- 资源查看器 → 工具栏 → "切换视图" → 选择三种之一
- 全局设置页 → 查看器默认 → 叶子文件夹自动进入阅读（开启/关闭）、章节列表默认展示模式（网格 / 列表）

**约束**：

- 切换模式后立即生效，下次打开该资源时记忆上次选择的模式
- 模式信息存储在 Resource 记录中（见 [@05-数据模型.md](05-数据模型.md)）

## 4. 功能清单

| ID | 功能点       | 优先级 | 说明                                                                             |
| -- | ------------ | ------ | -------------------------------------------------------------------------------- |
| E1 | 章节模式     | P0     | 子文件夹作为章节，先选章节再阅读；支持跨章节连续滑动；散落文件混入章节列表末尾（见 [@04-资源查看器.md](04-资源查看器.md)） |
| E2 | 平铺网格模式 | P1     | 当前层级网格展示，点击文件夹逐层进入，点击文件进入查看器；单层图片文件自动退化为全文件网格；可通过全局设置跳过网格直接进入阅读 |
| E3 | 画廊模式     | P2     | 递归展开所有子文件夹的全部文件到一个大网格；用户手动选择此模式                                       |
| E4 | 自动模式判定 | P1     | 添加资源时根据目录结构自动推荐（章节/平铺网格）                                                   |
| E5 | 手动模式切换 | P1     | 用户可在详情或查看器中切换模式                                                   |

## 5. 边界条件

1. **章节模式但子文件夹为空**：该章节在列表中置灰，标记"空章节"，不可点击
2. **平铺网格模式下点击子文件夹**：进入该子文件夹的平铺网格视图（逐层导航）
3. **画廊模式文件过多（>1000）**：P2 虚拟滚动优化，当前可接受性能限制
4. **混合模式边界**：文件夹内顶层同时有散落图片和子文件夹 → 判定为"平铺网格"（非章节模式）；顶层仅子文件夹（无散落图片）→ 章节模式；顶层仅图片（无子文件夹）→ 平铺网格（退化为全文件网格）
5. **空文件夹**：添加资源时自动跳过，首页不显示
6. **拆分资源**：拆出的子资源作为新资源，重新自动判定组织模式
7. **章节模式顶层有散落文件**：散落文件显示在章节列表末尾（章节在前、文件在后），可点击进入查看器
8. **画廊模式翻页范围**：进入查看器后左右滑动覆盖全画廊扁平列表中的所有兼容文件
9. **叶子文件夹自动进入**：资源仅有单层文件（无子文件夹）时，全局设置 `autoEnterLeafFolder` 开启则直接进入查看器翻页，关闭则先展示文件缩略图网格
