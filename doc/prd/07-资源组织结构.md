# 07 — 资源组织结构

> 所属模块：跨 A/D · 优先级：P0 · 状态：已定稿

---

## 1. 模块概述

一个资源（通常是一个文件夹）内部可能有不同的文件组织方式。用户在文件浏览器中将一个文件夹添加为资源后，系统需要知道该资源内部是"如何组织的"，以便在查看器中以正确的方式呈现内容。

本模块定义资源的四种内部组织模式，以及用户如何切换和管理它们。

---

## 2. 四种组织模式

### 模式 1：直接阅读（Direct Read）

> 文件夹内直接包含图片文件，一层结构，打开即翻阅。

**适用场景**：图包、画集、已解压的单行本漫画。

**目录结构示例**：

```
/海贼王_卷01/
  ├── 001.jpg
  ├── 002.jpg
  ├── 003.jpg
  └── ...
```

**查看器行为**：

- 进入后直接以图片查看器展示，按文件名排序
- 左上角显示资源名 + 页码

---

### 模式 2：章节模式（Chapter Mode）

> 文件夹内包含多个子文件夹（章节），每个子文件夹视为一个阅读单元。

**适用场景**：按章节/卷整理的漫画。

**目录结构示例**：

```
/海贼王/
  ├── 第001话_冒险的黎明/
  │   ├── 01.jpg
  │   ├── 02.jpg
  │   └── ...
  ├── 第002话_那个男人/
  │   ├── 01.jpg
  │   └── ...
  └── ...
```

**查看器行为**：

- 进入后先展示章节列表（每个子文件夹作为一个章节项）
- 章节项显示：章节封面缩略图（子文件夹第一张图）+ 章节名
- 点击章节 → 进入统一查看器翻阅该章节
- 章节之间支持跨章节连续阅读：末页继续左滑提示"下一章: xxx"，松手进入下一章首页（见 [@04-资源查看器.md](04-资源查看器.md) 4.1.4）

**章节排序**：按子文件夹名称排序（支持自然排序，即"第2话"排在"第10话"前面）

---

### 模式 3：平铺网格（Flat Grid）

> 以网格展示当前目录下的所有文件夹和兼容文件。与文件浏览器网格视图体验一致。

**适用场景**：混合内容目录、杂项收藏夹。

**目录结构示例**：

```
/杂项收藏/
  ├── cosplay合集.jpg
  ├── 设定集.pdf
  ├── 花絮.mp4
  ├── 画集/
  │   ├── 01.jpg
  │   └── 02.jpg
  └── ...
```

**查看器行为**：

- 进入后以网格展示当前层级的所有文件夹和兼容文件
- 文件夹显示封面缩略图 + 名称，文件显示自身缩略图 + 名称
- 点击文件 → 进入统一查看器，左右滑动浏览**当前平铺网格**中的兼容文件
- 点击文件夹 → 进入该子文件夹的平铺网格视图（逐层进入，非递归展开）
- 返回键 → 返回上一级目录的平铺网格

---

### 模式 4：画廊模式（Gallery）

> **递归展开**所有子文件夹的全部兼容文件到一个大网格中，打破层级，一屏浏览所有内容。

**适用场景**：从各种渠道收集的散图、没有章节概念的无序图包。

**目录结构示例**：

```
/散图合集/
  ├── 子文件夹A/
  │     ├── 01.jpg
  │     └── 02.jpg
  ├── 子文件夹B/
  │     └── 03.jpg
  ├── standalone.jpg
  └── doc.pdf
```

**查看器行为**：

```
进入后平铺展开为：

┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│ 子文件夹A │ │ 子文件夹A │ │ 子文件夹B │ │standalone│ │  doc     │
│  01.jpg  │ │  02.jpg  │ │  03.jpg  │ │  .jpg   │ │  .pdf   │
└──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘
```

- **展开规则**：递归遍历所有子文件夹，收集全部兼容文件，平铺到一个网格
- **排序**：默认按文件夹名 → 文件名（字母升序）排列
- **点击文件** → 进入统一查看器，左右滑动浏览**画廊中所有文件的扁平列表**
- 回到画廊页面：顶栏显示资源名 + 总文件数
- 大量文件时（>1000）：P2 虚拟滚动

**画廊模式排序示例**：

```
子文件夹A/01.jpg
子文件夹A/02.jpg
子文件夹B/03.jpg
standalone.jpg
doc.pdf
```

**约束**：

- 画廊模式不保留文件夹层级信息——所有文件视为平面列表
- 子文件夹过多（>100 个）时，画廊模式可能加载缓慢，提示用户考虑使用平铺网格模式

## 3. 模式选择

### 3.1 默认行为（自动判定）

系统根据文件夹结构**自动推荐**模式，用户可手动修改。判定按以下顺序**从上到下**匹配，命中即停止（行间互斥）：

| 顺序 | 条件                                                    | 推荐模式 | 判定逻辑             |
| ---- | ------------------------------------------------------- | -------- | -------------------- |
| 1 | 文件夹内直接包含 ≥1 张图片，**无子文件夹**              | 直接阅读 | 图片为主，一层结构   |
| 2 | 文件夹内含 ≥1 个子文件夹，且**至少一个子文件夹内含图片**，且顶层**无混合文件类型**（顶层仅有子文件夹，无散落图片/PDF/视频） | 章节模式 | 存在明显章节/卷层级  |
| 3 | 文件夹内含混合文件类型（顶层有图片 + PDF + 视频等散落文件）+ 子文件夹 | 平铺网格 | 混合结构，按层级导航 |
| 4 | 无法判定（空文件夹/无识别文件）                         | 平铺网格 | 兜底方案             |

**歧义消解规则**：
- 顶层有散落图片 + 子文件夹（子文件夹内有图片）→ 命中顺序 3（平铺网格），不命中顺序 2
- 顶层仅子文件夹（无散落图片），子文件夹内有图片 → 命中顺序 2（章节模式）
- 即"顶层是否有散落图片/文件"是区分章节模式与平铺网格的关键

### 3.2 用户手动切换

**入口**：

- 资源详情弹窗 → "组织模式" 选项 → 选择四种之一
- 资源查看器 → 工具栏 → "切换视图" → 选择四种之一

**约束**：

- 切换模式后立即生效，下次打开该资源时记忆上次选择的模式
- 模式信息存储在 Resource 记录中（见 [@05-数据模型.md](05-数据模型.md)）

## 4. 功能清单

| ID | 功能点       | 优先级 | 说明                                                                             |
| -- | ------------ | ------ | -------------------------------------------------------------------------------- |
| E1 | 直接阅读模式 | P0     | 基础模式，进入后直接图片翻页                                                     |
| E2 | 章节模式     | P0     | 子文件夹作为章节，先选章节再阅读；支持跨章节连续滑动（见 [@04-资源查看器.md](04-资源查看器.md)） |
| E3 | 平铺网格模式 | P1     | 当前层级网格展示，点击文件夹逐层进入，点击文件进入查看器                         |
| E4 | 画廊模式     | P2     | 递归展开所有子文件夹的全部文件到一个大网格                                       |
| E5 | 自动模式判定 | P1     | 添加资源时根据目录结构自动推荐                                                   |
| E6 | 手动模式切换 | P1     | 用户可在详情或查看器中切换模式                                                   |

## 5. 边界条件

1. **章节模式但子文件夹为空**：该章节在列表中置灰，标记"空章节"，不可点击
2. **直接阅读模式但文件夹内无图片**：不显示在首页，资源被视为无效直接跳过
3. **平铺网格模式下点击子文件夹**：进入该子文件夹的平铺网格视图（逐层导航）
4. **画廊模式文件过多（>1000）**：P2 虚拟滚动优化，当前可接受性能限制
5. **混合模式边界**：文件夹内顶层同时有散落图片和子文件夹 → 判定为"平铺网格"（非章节模式，也不直接阅读）；顶层仅子文件夹（无散落图片）→ 章节模式
6. **单张图片文件夹**：直接阅读模式，翻页按钮不可用
7. **空文件夹**：添加资源时自动跳过，首页不显示
8. **拆分资源**：拆出的子资源作为新资源，重新自动判定组织模式
