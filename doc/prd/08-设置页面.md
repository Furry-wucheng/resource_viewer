# 08 — 设置页面

> 所属模块：跨模块 · 优先级：P0 · 版本：v1.2 · 日期：2026-06-17 · 状态：已定稿

---

## 1. 模块概述

设置页面作为第三个底部 Tab，统一管理应用的所有可配置项，包括缓存管理、外观设置、查看器默认行为、数据源同步策略等。

---

## 2. 入口

- **主导航**：底部导航第三个 Tab（与首页库、数据源并列）
- **快捷入口**：查看器工具栏设置按钮（⚙）→ 跳转到设置页对应分组

---

## 4. 详细需求

### 4.1 缓存管理

#### 4.1.1 信息展示

```
┌──────────────────────────────────────┐
│  缓存管理                            │
├──────────────────────────────────────┤
│  缩略图缓存                          │
│  已用：1200 M / 上限：2000 M         │
│  ═══════════●═══════ 60%            │  ← 进度条
│                                      │
│  容量上限                            │
│  [500M] [1000M] [1500M] [2000M] [自定义]│  ← 选项组
│                                      │
│  [清理缩略图缓存]                    │  ← 按钮
│                                      │
│  缓存位置：/data/cache/thumbnails/   │  ← 只读
└──────────────────────────────────────┘
```

#### 4.1.2 缓存策略

- **淘汰策略**：LRU（最近最少使用），超出容量上限时自动清理最久未访问的缩略图
- **默认容量上限**：500 M
- **容量上限可调**：500 M / 1000 M / 1500 M / 2000 M / 自定义（输入整数 M）

#### 4.1.3 手动清理

- 点击"清理缩略图缓存"按钮
- 二次确认弹窗："将清除 1200 M 的缩略图缓存，下次浏览时需重新生成。确定吗？"
- 确认后删除所有本地缩略图文件
- 清理后首页缩略图将重新生成（不影响资源数据）
- 清理过程中显示进度

### 4.2 外观设置

| 设置项   | 选项                   | 默认值   |
| -------- | ---------------------- | -------- |
| 深色模式 | 跟随系统 / 深色 / 浅色 | 跟随系统 |

- 仅改变 UI 主题色（底部导航栏、设置页、弹窗等），查看器背景始终为黑色，不受此设置影响

### 4.3 查看器默认设置

| 设置项         | 选项                       | 默认值 | 说明                                                                 |
| -------------- | -------------------------- | ------ | -------------------------------------------------------------------- |
| 默认翻页方向   | 左→右 / 右→左            | 右→左 | 日漫默认右到左阅读；可在查看器内临时切换                             |
| 双页显示       | 自动 / 始终单页 / 始终双页 | 自动   | 自动 = 宽度≥900dp 时双页；始终双页 = 忽略宽度；在查看器内可临时切换 |
| 跨章节连续阅读 | 开启 / 关闭                | 开启   | 关闭后章节末页不再提示进入下一章                                     |

### 4.4 数据源自动同步（P2）

| 设置项       | 选项                             | 默认值 |
| ------------ | -------------------------------- | ------ |
| 自动同步间隔 | 关闭 / 每小时 / 每 6 小时 / 每天 | 关闭   |

- 开启后按间隔检查所有启用的数据源
- 仅检测文件变更（新增/删除/修改），同步节流避免 SMB 流量
- 同步结果：成功无提示，失败静默记录日志

### 4.6 恢复默认设置

- 入口：设置页底部"恢复默认设置"按钮
- 点击后二次确认弹窗："将重置所有设置为默认值，确定吗？"
- 确认后将 AppConfig 所有字段重置为默认值（见 `@05-数据模型.md` §2.5）
- 重置后立即生效，UI 反映新主题/翻页方向等

---

## 5. 边界条件

1. **缓存清理中退出设置页**：后台继续清理，完成后无提示
2. **缓存容量设为极小值（如 100 M）**：不接受小于 500 M 的值，提示"最小容量为 500 M"
3. **缓存路径不可用**：使用应用默认缓存目录兜底
4. **设置项持久化**：所有设置变更立即写入本地存储，下次启动恢复
