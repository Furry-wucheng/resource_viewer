## 1. 章节画廊模式

### 1.1 背景

现有组织模式覆盖了三类阅读方式：

- **章节模式**：资源根目录下的子文件夹作为章节，进入章节后只阅读该章节当前层级的文件。
- **平铺网格**：按当前层级浏览，文件夹逐层下钻。
- **画廊模式**：递归展开整个资源下的全部支持文件，作为一个全局扁平列表。

部分资源的结构介于章节和画廊之间：根层仍然有明确章节，但每个章节内部可能继续按页组、子文件夹、版本或图片包分层。用户希望保留章节入口，同时在进入章节后不再逐层下钻，而是直接阅读该章节下全部支持文件。

### 1.2 命名

新增组织模式命名为：**章节画廊模式**。

英文/代码命名：`chapterGallery`。

含义：根层按章节组织，章内按画廊方式递归扁平化。

### 1.3 用户故事

作为资源查看用户，我希望一个资源可以按章节进入；进入某一章后，应用自动穿透该章内所有子目录，把所有支持文件组成连续阅读序列，这样我不需要在章节内部反复下钻文件夹。

### 1.4 功能规则

- 资源根层展示方式与章节模式一致：根目录下的子文件夹显示为章节。
- 章节列表中的章节封面取该章节递归范围内第一张支持图片。
- 章节页数/文件数统计取该章节递归范围内全部支持文件数量。
- 点击章节后，递归收集该章节路径下所有支持文件，按目录路径、文件名排序后进入统一查看器。
- 查看器翻页范围仅限当前章节递归得到的扁平文件列表，不跨到其他章节。
- 开启跨章节连续阅读设置时，当前章节最后一页继续翻阅进入下一章第一页；当前章节第一页反向翻阅进入上一章最后一页。
- 跨章节跳转提示沿用章节模式文案：`下一章: xxx` / `上一章: xxx` / `已是最后一章` / `已是第一章`。
- 顶层散落文件仍按章节模式现有规则展示在章节列表末尾。
- 该模式不参与自动判定，仅由用户手动选择。
- 当资源根目录没有子文件夹时，该模式不可选或自动回退到平铺网格。

### 1.5 支持文件

沿用现有兼容文件集合：

- 图片：jpg / jpeg / png / gif / webp / bmp
- PDF：pdf
- 视频：mp4 / mkv / avi / mov / wmv / flv / webm / m4v
- 压缩包：zip / rar / 7z

### 1.6 验收标准

- [ ] 模式选择器出现“章节画廊”选项，查看器工具栏可用短标签“章廊”。
- [ ] 资源详情弹窗可将资源切换为章节画廊模式。
- [ ] 章节画廊模式打开资源时进入章节列表页。
- [ ] 章节列表展示根目录直接子文件夹作为章节。
- [ ] 章节封面和计数来自章节目录的递归支持文件。
- [ ] 点击章节后直接进入查看器，内容为该章节下递归扁平后的支持文件序列。
- [ ] 章节画廊模式支持与章节模式一致的跨章节连续阅读。
- [ ] 从当前章最后一页继续翻阅进入下一章第一页。
- [ ] 从当前章第一页反向翻阅进入上一章最后一页。
- [ ] 章内嵌套文件夹不会作为中间页面出现。
- [ ] 无子文件夹资源不能使用章节画廊模式，历史无效值回退到平铺网格。
