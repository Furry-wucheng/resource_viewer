## 3. 逐项详细分析

### 3.1 SMB 网络共享 — `dart_smb2`

> 对应 [@prd/03-数据源管理.md](../prd/03-数据源管理.md)

| 属性 | 详情 |
|------|------|
| 版本 | 0.1.0（2026-06 发布） |
| 协议 | SMB 2.0.2 ~ 3.1.1（含签名/加密） |
| 平台 | Android, iOS, Linux, macOS, Windows |
| 机制 | 纯 Dart FFI，底层 libsmb2 v6.1.0（成熟 C 库） |
| 许可证 | BSD-3-Clause |

**PRD 需求覆盖**：

| PRD 需求 | API 映射 |
|----------|---------|
| 添加 SMB 源（地址+凭据+测试连接） | `Smb2Pool.connect(host:, share:, user:, password:, domain:)` + `echo()` |
| 文件浏览器目录列表 | `listDirectory(path)` |
| 文件元数据 | `stat(path)` |
| 连接失败检测（源不可达） | `Smb2Exception` + `Smb2ErrorType` 枚举 |
| 连接恢复 | `Smb2Pool` 自动重连 |

**备选方案**：`smb_connect` (v0.0.9)，纯 Dart 实现但仅支持 SMB 1.0~2.1，不支持 SMB 3.x 加密，已 16 个月未维护。不推荐。

**缓解措施**：BSD 协议可 fork；底层 `libsmb2` 成熟稳定；备选自定义 FFI 或平台通道方案（已有 R-Shop / CB-File-Hub 参考实现）。

### 3.2 PDF 渲染 — `pdfrx`

> 对应 [@prd/04-资源查看器.md](../prd/04-资源查看器.md)

| 属性 | 详情 |
|------|------|
| 版本 | 2.4.4（活跃维护） |
| 平台 | Android, iOS, Linux, macOS, Windows, Web |
| 引擎 | PDFium（Chrome 同款） |
| 许可证 | MIT |

**选型对比**：

| | pdfrx | pdfx | syncfusion |
|------|-------|------|------------|
| Windows | ✅ | ✅ | ✅ |
| macOS | ✅ | ✅ | ✅ |
| Linux | ✅ | ❌ | ✅ |
| 加密 PDF | ✅ | ❌ | ✅ |
| 页面→图片 API | ✅ | ✅ | 部分 |
| 多线程渲染 | ✅ | ❌ | 未知 |
| 许可证 | MIT | MIT | 商业 |

**关键 API**：

```dart
// 打开 PDF 文档
final document = await PdfDocument.openFile(filePath,
  passwordProvider: createSimplePasswordProvider('password'),
);

// 单页渲染为图片
final page = await document.getPage(1);
final image = await page.render(width: w, height: h);
// image → PdfImageProvider → Image widgets（返回 ImageProvider，非直接 bytes）
```

**注意**：PDFium 通过 native assets/DylibBundle 自动打包，无需手动配置或开发者模式。

### 3.3 压缩包阅读

> 对应 [@prd/04-资源查看器.md](../prd/04-资源查看器.md) §4.5

#### ZIP — `archive` ✅

| 属性 | 详情 |
|------|------|
| 版本 | 4.0.9 |
| 下载量 | 9.79M |
| 平台 | 全部 6 平台 |
| 许可证 | MIT |

```dart
// 内存解压，不写磁盘
final archive = ZipDecoder().decodeBytes(bytes, password: '...');
for (final file in archive) {
  if (file.isFile && file.name.endsWith('.jpg')) {
    final imageBytes = file.readBytes(); // 内存解压
  }
}
```

支持 ZIP AES-256 加密、ZipCrypto 传统加密。

#### RAR — `unrar` ⚠️

| 属性 | 详情 |
|------|------|
| 版本 | 0.1.2 |
| 下载量 | 49 |
| 平台 | Android, iOS, Linux, macOS, Windows |

基于官方 UnRAR 库的 FFI 封装。支持 Windows，声称支持"提取到内存或磁盘"。但极不成熟（49 下载，未经验证的发布者）。**P2 再实现，优先 ZIP。**

#### 7z — ❌ 不可行

pub.dev 无可用 7z 容器解析库。`flutter_lzma` 仅支持 Android/iOS 且已 3 年未更新。`archive` 包的 XZ 解码器仅处理单文件压缩，不解 7z 容器。

**后续选项**：捆绑 7-Zip CLI 通过进程调用；自定义 FFI 封装 7z SDK；或直接不列入需求。

### 3.4 视频播放 — `media_kit`

> 对应 [@prd/04-资源查看器.md](../prd/04-资源查看器.md) §4.4

| 属性 | 详情 |
|------|------|
| 核心 | `media_kit: ^1.2.6` |
| 渲染 | `media_kit_video: ^2.0.1` |
| 原生库 | `media_kit_libs_video: ^1.0.7`（FFmpeg/libmpv） |
| 平台 | 全部 6 平台（含 Windows 7+） |
| 许可证 | MIT |

**`video_player` 官方包不支持 Windows**，`media_kit` 是唯一选择。

**跨源播放约束**：media_kit 接收本地路径或 URL，不消费应用内 `FileSource.streamFile()`。因此本地源视频可直接 `Media(localPath)`，SMB/未来网络源视频必须通过应用内 `127.0.0.1` HTTP Range 代理播放，由代理把播放器的 Range 请求转换为 `FileSource.readRange()`。`dart_smb2` 已提供 `Smb2Pool.readFileRange`，可支撑拖动进度条后的随机读取；禁止把 `\\host\share\file.mp4` 或 `smb://host/share/file.mp4` 直接交给 media_kit 作为通用方案，因为这会绕过 SecureStorage 凭据和 `SmbFileSource` 连接池。

**PRD 需求覆盖**：

| PRD 需求 | 实现方式 |
|----------|---------|
| 单击显隐工具栏 | `GestureDetector` + 自定义控件叠加层 |
| 双击暂停/播放 | `player.playOrPause()` + `onDoubleTap` |
| 长按倍速 | `player.setRate(n)` + 自实现手势 |
| 进度条 | 自定义控件或 `MaterialDesktopVideoControls`；SMB 视频 seek 依赖 HTTP Range 代理 |
| 键盘快捷键 | 内置（空格暂停、方向键快进、ESC 全屏等） |

**注意**：FFmpeg 库增加包体约 50-80MB。长按倍速手势桌面端体验需优化（建议右键+滚轮辅助）。

### 3.5 本地数据库 — `drift`

> 数据模型详见 [@prd/05-数据模型.md](../prd/05-数据模型.md)

| 属性 | 详情 |
|------|------|
| 版本 | 2.34.0（活跃维护） |
| 类型 | SQLite ORM（代码生成 + 类型安全） |
| 平台 | 全部 6 平台 |
| 许可证 | MIT |

**选型对比**：

| | drift | sqflite | sqflite_common_ffi | Isar |
|------|-------|---------|-------------------|------|
| Windows | ✅ | ❌ | ✅ | ❌ |
| 类型安全 | ✅ 代码生成 | ❌ 手动 Map | ❌ 手动 Map | ✅ |
| 反应式 Stream | ✅ | ❌ | ❌ | ✅ |
| 级联删除 | ✅ | 手动 | 手动 | N/A |
| 维护状态 | ✅ 活跃 | ✅ 活跃 | ✅ 活跃 | ❌ 停滞 |

**数据模型映射**：

```
Source (1) ──→ (N) Resource (N) ──→ (M) Tag
                       │                    │
                       └── ResourceTag ────┘  (junction)
```

- 级联删除：删 Source → 自动删 Resource → 自动删 ResourceTag；Tag 保留
- 标签交集筛选：`SELECT resourceId FROM ResourceTag WHERE tagId=A INTERSECT SELECT ... WHERE tagId=B`

**SMB 密码存储**：不存入数据库。使用 `flutter_secure_storage`（Windows DPAPI 加密），Source 表仅存 `passwordStored: bool` 标记。

### 3.6 缩略图生成策略

> 对应 [@prd/01-资源库首页.md](../prd/01-资源库首页.md) §4.2 · [@prd/08-设置页面.md](../prd/08-设置页面.md) §4.1

| 来源类型 | 方案 | 包 |
|----------|------|-----|
| 图片文件（jpg/png/webp/gif） | 解码 + 缩放至 2:3 竖版尺寸 | `image` |
| PDF 首页 | 渲染 page 1 → Uint8List | `pdfrx` |
| ZIP 首图 | 读取 ZIP 内第一张图片字节 | `archive` |
| 视频首帧 | 截图功能 | `media_kit`（`screenshot()`） |
| 未知类型 | 通用占位图 | 内嵌资源 |

**缓存策略**：本地目录存储（`path_provider` 获取路径），LRU 淘汰，默认上限 500MB，用户可调。

## 5. 风险矩阵

| 风险 | 等级 | 影响 | 缓解措施 |
|------|------|------|---------|
| `dart_smb2` v0.1.0 稳定性未知 | 🟡 中 | SMB 功能不可用 | BSD 协议可 fork；libsmb2 底层成熟；备选自定义 FFI |
| `unrar` RAR 支持不成熟 | 🔴 高 | RAR 无法阅读 | P2 实现；先仅支持 ZIP；必要时回退磁盘解压 |
| 7z 无可用 Dart 库 | 🔴 高 | 7z 无法阅读 | 不列入 MVP；后续评估外部 CLI 方案 |
| `pdfrx` PDFium native 打包 | 🟢 低 | 构建配置 | native assets 自动打包，无需手动配置 |
| FFmpeg 库增大包体 50-80MB | 🟢 低 | 安装包体积 | 可选精简库；桌面应用体积容忍度高 |
| 长按倍速桌面 UX 不佳 | 🟡 中 | 视频体验差 | 桌面端用右键+滚轮或键盘快捷键辅助 |

| 阶段 | 内容 | 验证的核心依赖 |
|------|------|--------------|
| **P0 基础** | 数据模型 + 本地源 + 直接阅读 + 图片查看器 + 标签 CRUD | drift, image, file_picker |

| **P2 扩展** | 压缩包阅读 + 自动同步 | unrar |

## 7. 架构约束

1. Source 类型（local/smb/ftp/webdav）使用**策略模式/接口抽象**，为 FTP/WebDAV 预留扩展点
2. ViewerPage 需区分两种进入上下文：首页（遵守组织模式） vs 文件浏览器（平级文件滑动）
3. SMB 密码只在 `flutter_secure_storage` 中存储，Source 表不存明文/密文密码
4. 源不可达时 Resource.isAvailable = false（不删除，保留标签关联）
5. 缩略图缓存独立目录，LRU 淘汰，不污染数据库
6. 文件浏览器排序独立于每级目录，默认"文件夹优先+文件名升序"

---

## 8. 结论

**整体技术可行性：高。** Flutter Desktop + Dart FFI 生态已可覆盖除 7z 压缩包外的所有核心需求。关键风险集中在 RAR/7z 和一些新包（dart_smb2、unrar）的成熟度上，均有合理的降级或延后策略。项目可以启动开发。
