## 1. 架构总览

采用 Flutter 官方推荐的**分层 MVVM + Repository 模式**，三层分离关注点：

```
┌─────────────────────────────────────────────────┐
│  UI 层 (Presentation)                            │
│  Views (Widget) ←→ ViewModels (ChangeNotifier)   │
├─────────────────────────────────────────────────┤
│  Domain 层                                       │
│  Models (freezed) + Use Cases (复杂业务逻辑)      │
├─────────────────────────────────────────────────┤
│  Data 层                                         │
│  Repositories (单一真相源) ← Services (外部 API)  │
└─────────────────────────────────────────────────┘
```

**原则**：

- UI 层只做渲染和路由，所有状态从 ViewModel 传入
- ViewModel 通过 `ChangeNotifier` 暴露不可变状态，注入 Repository
- Service 无状态，封装外部 API（数据库、SMB、PDF、压缩包等），返回原始模型
- Repository 消费一个或多个 Service，转换为 Domain Model，处理缓存与重试
- 复杂业务逻辑（跨仓库协作、资源扫描、组织模式判定）抽取为 Use Case

## 2. 技术选型

| 维度 | 方案 | 理由 |
|------|------|------|
| 状态管理 | ChangeNotifier + Provider | Flutter 官方推荐，轻量内置，学习成本低 |
| 路由 | go_router (`StatefulShellRoute.indexedStack`) | 声明式 URL 路由，保持 Tab 状态，桌面端深链接友好 |
| 领域模型 | freezed | 代码生成 immutable 类、copyWith、联合类型、JSON 序列化 |
| 数据库 | drift | SQLite ORM，类型安全，反应式 Stream，级联删除 |
| 凭据存储 | flutter_secure_storage | OS 级加密（Windows DPAPI），与数据库分离 |
| 轻量 UI 偏好 | SharedPreferences | 视图模式等无需数据库的用户偏好，KV 存取，零 schema 迁移 |

详见 [@01-技术可行性分析.md](01-技术可行性分析.md)。

## 4. 关键抽象设计

### 4.1 FileSource 接口（数据源策略模式）

统一本地/SMB/未来协议（FTP/WebDAV）的文件访问，为多源扩展预留：

```dart
abstract class FileSource {
  String get sourceId;
  Future<List<FileEntry>> listDirectory(String relativePath);
  Future<FileEntry?> stat(String relativePath);
  Future<Uint8List> readFile(String relativePath);
  Stream<Uint8List> streamFile(String relativePath);   // 大文件流式
  Future<bool> testConnection();
  Future<void> disconnect();
}

// 实现：
//   LocalFileSource (dart:io)
//   SmbFileSource   (dart_smb2)
//   FtpFileSource   (P2 扩展)
//   WebDavFileSource(P3 扩展)

// 工厂：FileSourceFactory.create(Source) → 按 type 返回对应实现，缓存活跃连接
```

### 4.2 ContentProvider 接口（查看器内容抽象）

统一图片/PDF/压缩包的翻页体验，使 ViewerPage 不感知底层格式。**视频不走此接口**，由 ViewerViewModel 直接管理 media_kit Player（见下方"视频混合浏览"）。

```dart
abstract class ContentProvider {
  int get pageCount;
  Future<Uint8List> loadPage(int index);   // → Image.memory()
  Future<void> dispose();
}

// ImageFolderProvider : 图片文件夹按文件名排序
// PdfProvider        : pdfrx 逐页渲染
// ArchiveProvider    : archive 包内图片列表 (P1)
```

#### 4.2.1 视频混合浏览（ViewerItem 联合类型）

图片/PDF/视频在同一 ViewerPage 内混合浏览，左右切换无缝。`FileSequenceViewerPage` 将 `List<FileEntry>` 转换为统一的 `List<ViewerMediaItem>`（见 `lib/shared/content_provider/viewer_media_item.dart`），ViewerViewModel 按当前页类型分发渲染：

```dart
// 已有代码，位于 lib/shared/content_provider/viewer_media_item.dart
@freezed
sealed class ViewerMediaItem with _$ViewerMediaItem {
  const factory ViewerMediaItem.image({required String title, required Future<Uint8List> Function() loadImage}) = _ImageItem;
  const factory ViewerMediaItem.video({required String title, required String videoPath}) = _VideoItem;
}
```

- **图片页** → `ContentProvider.loadPage(index)` → `Uint8List` → `Image.memory()`（或 `ViewerMediaItem.loadImage()` 直接返回字节）
- **视频页** → ViewerViewModel 直接创建 media_kit `Player` + `VideoController`，由 `VideoPlayer` widget 渲染（流式解码，不参与预加载窗口机制）

**ViewerViewModel 渲染分发**：

```dart
Widget buildPage(ViewerItem item) {
  return switch (item) {
    ImagePageItem(:final pageIndex) =>
      ExtendedImage(imageProvider: _provider.loadPage(pageIndex)),
    PdfPageItem(:final pageIndex) =>
      ExtendedImage(imageProvider: _provider.loadPage(pageIndex)),
    VideoItem(:final filePath) =>
      VideoPlayer(controller: _videoControllerFor(filePath)),
  };
}
```

**关键约定**：
- 预加载窗口（当前+前1+后2）仅对图片/PDF 页生效，遇到视频页跳过预加载
- 双页并排仅对图片/PDF 页生效，遇到视频页自动回退单页
- 错误处理分流：图片/PDF 页用 `MediaLoadError`+占位重试，视频页解码失败单独处理（仍走 `MediaLoadError`，mediaType=`video`）
- ContentProvider 接口保持纯粹（`loadPage` 返回 `Uint8List`），视频不污染此接口


### 4.3 OrganizationStrategy（组织模式策略）

对应 [@prd/07-资源组织结构.md](../prd/07-资源组织结构.md) 三种模式：

```dart
abstract class OrganizationStrategy {
  OrganizationMode get mode;
  Future<List<Chapter>> getChapters(Resource r, FileSource source);
  Future<List<FileEntry>> getContents(Resource r, FileSource source);
  ContentProvider createProvider(Resource r, FileSource source, Chapter? chapter);
}

// ChapterStrategy   : 章节模式，子文件夹作为章节
// FlatGridStrategy  : 平铺网格，逐层导航（含原"直接阅读"场景，单层图片文件自然退化为全文件网格）
// GalleryStrategy   : 画廊模式，递归扁平化 (P2)
```

>`getChapters` 和 `getContents` 返回 `Future`，因为内部需要调用 `FileSource.listDirectory`（异步）获取目录结构。`createProvider` 仍同步返回 `ContentProvider` 实例（其 `loadPage` 内部异步加载）。`getContents` 产出统一的 `List<FileEntry>`，各策略按自身逻辑填充（FlatGrid: 当前层级文件夹+文件，含单层文件退化为全文件网格；Gallery: 递归扁平全部文件）。

### 4.4 ThumbnailGenerator（缩略图策略）

```dart
abstract class ThumbnailGenerator {
  Future<String?> generate(FileSource source, String relativePath);
}

// 按 Resource.type 路由到对应生成器：
//   ImageThumbnailGenerator   (image 包解码缩放)
//   PdfThumbnailGenerator     (pdfrx 渲染第一页)
//   ArchiveThumbnailGenerator (archive 读取首图, P1)
//   VideoThumbnailGenerator   (media_kit 截图)
```

## 5. 路由设计（go_router）

```text
StatefulShellRoute.indexedStack        # 3 Tab 状态保持
├── Branch 0: /home                    # 首页
│   └── GoRoute: /home
├── Branch 1: /sources                 # 数据源
│   ├── GoRoute: /sources              # 源列表
│   └── GoRoute: /sources/:id/browser  # 文件浏览器
└── Branch 2: /settings                # 设置

GoRoute: /viewer/:resourceId           # 全屏查看器 (parentNavigatorKey 跳出 Tab)
GoRoute: /viewer/file/:sourceId        # 文件浏览器来源查看器
GoRoute: /tags/manager                 # 标签管理 (全屏)
```

**要点**：

- `StatefulShellRoute.indexedStack` 保持各 Tab 滚动位置和状态（对应 [@prd/06-页面结构与交互.md](../prd/06-页面结构与交互.md) 4.1 "首页 → 查看器 → 返回恢复滚动位置"）
- 查看器使用 `parentNavigatorKey` 推到根导航栈，覆盖 Tab 栏实现全屏沉浸
- ResourcePicker / 标签编辑作为 `showDialog` 弹窗，不走路由

## 6. 依赖注入（Provider）

```dart
// app.dart
MultiProvider(
  providers: [
    // Services (单例)
    Provider(create: (_) => DatabaseService()),
    Provider(create: (_) => SecureStorageService()),
    Provider(create: (_) => PdfRenderService()),
    Provider(create: (_) => ArchiveService()),

    // Repositories (依赖 Services)
    Provider(create: (ctx) => SourceRepository(ctx.read<DatabaseService>())),
    Provider(create: (ctx) => ResourceRepository(ctx.read<DatabaseService>())),
    Provider(create: (ctx) => TagRepository(ctx.read<DatabaseService>())),
    Provider(create: (ctx) => FilesystemRepository(ctx.read<DatabaseService>())),
    Provider(create: (ctx) => ThumbnailRepository(ctx.read<DatabaseService>())),

    // Use Cases (依赖 Repositories，显式指定类型避免误注入)
    Provider(create: (ctx) => ScanResourcesUseCase(
      ctx.read<ResourceRepository>(),
      ctx.read<TagRepository>(),
    )),
    Provider(create: (ctx) => DetectOrganizationModeUseCase(
      ctx.read<FilesystemRepository>(),
    )),
    Provider(create: (ctx) => FilterResourcesByTagsUseCase(
      ctx.read<TagRepository>(),
    )),
    ...
  ],
  child: MaterialApp.router(routerConfig: router),
)
```

ViewModel 在各 Page 内通过 `ChangeNotifierProvider` 创建，注入所需的 Repository/UseCase。

## 7. 数据流示例

### 7.1 首页标签筛选

```
用户点击标签 → HomeViewModel.selectTag(tagId)
  → FilterResourcesByTagsUseCase.execute(selectedTagIds)
    → TagRepository → drift INTERSECT 查询
  → 返回 List<Resource> (freezed)
  → ViewModel.notifyListeners()
  → ResourceGrid 重建
```

### 7.2 SMB 文件浏览

```
FileBrowserViewModel.openDirectory(path)
  → FilesystemRepository.listDirectory(sourceId, path)
    → FileSourceFactory.create(source) → SmbFileSource
    → SmbFileSource.listDirectory(path) → dart_smb2 API
  → 返回 List<FileEntry> (freezed, 统一跨源)
  → ViewModel.notifyListeners()
```

### 7.3 进入资源查看器

```
点击资源 → 跳转 /viewer/:resourceId
  → ViewerViewModel.initialize(resourceId)
    → ResourceRepository.getResource(id) → Resource (freezed)
    → FileSourceFactory.create(source) → FileSource
    → OrganizationStrategyFactory.create(resource.organizationMode)
    → strategy.getContents(resource, fileSource) → List<FileEntry>
      （文件夹/图片/PDF/视频等兼容文件条目）
    → 图片/PDF 页：strategy.createProvider(...) → ContentProvider
    → 画廊/平铺网格模式：FileEntry 列表 → FileSequenceViewerPage → ViewerPage.media()
    → 视频页：ViewerViewModel 为每个 VideoItem 创建 media_kit Player
  → ViewerPage 渲染，按当前 ViewerItem 类型分发：
    - ImagePageItem/PdfPageItem → provider.loadPage(index) → Image.memory
    - VideoItem → VideoPlayer(controller)
```

### 7.4 扫描入库（后台异步）

```
ResourcePicker 确认 → BatchAddResourcesUseCase.execute(entries)
  → 在 Isolate 中遍历：
    ① 过滤已入库项 (sourceId + relativePath 去重)
    ② 创建 Resource 记录
    ③ DetectOrganizationModeUseCase 判定模式
    ④ ThumbnailGenerator 生成缩略图
    ⑤ 通过 Stream 推送进度
  → 首页 ViewModel 监听进度流，实时刷新网格
```

## 8. 跨层关注点

| 关注点 | 方案 |
|--------|------|
| 错误处理 | Repository 层捕获并转为 `Result<T>` 包装；ViewModel 转为 UI 错误状态 |
| 缩略图缓存 | ThumbnailRepository 管理 LRU，磁盘存储，容量上限由设置页配置（默认 500MB） |
| SMB 密码 | SecureStorageService 存取，Source 表仅存 `passwordStored: bool` 标记，UI 脱敏 `****` |
| 连接池 | FileSourceFactory 缓存活跃 FileSource 连接；源禁用/删除时 disconnect |
| 异步扫描 | Isolate 后台执行，进度通过 Stream 通知 ViewModel，避免 UI 卡顿 |
| 数据一致性 | drift Stream 查询驱动 UI 自动更新；Tab 切换时静默刷新 |
| 平台适配 | UI 层根据 `MediaQuery` / `LayoutBuilder` 自适应布局（窄屏/宽屏分支） |

## 11. 约束与注意事项

1. **View 不持有业务状态**：所有状态来自 ViewModel，View 只负责渲染和派发用户事件
2. **Repository 是单一真相源**：ViewModel 不直接访问 Service，必须经 Repository。**例外**：轻量级表单校验（空值/长度/重名）允许在 ViewModel 内执行，不经过 Repository；涉及持久化一致性校验仍走 Repository
3. **Domain Model 不可变**：使用 freezed 生成，跨层传递安全；数据库模型不直接暴露给 UI
4. **FileSource 连接生命周期**：源删除时必须 disconnect（对应 [@prd/03](../prd/03-数据源管理.md) §4.7），避免 SMB 连接泄漏
5. **ContentProvider 必须释放**：查看器退出时调用 `dispose()`（对应 [@prd/04](../prd/04-资源查看器.md) §5 第 7 条），清理 PDF 文档句柄和图片缓存
6. **缩略图生成在后台**：避免阻塞 UI，通过 Stream 推送结果
7. **新增数据源类型**：只需实现 FileSource 接口 + 在 FileSourceFactory 注册，不改上层代码
