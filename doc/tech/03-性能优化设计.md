## 1. 概述

本文档针对 PRD 中识别的性能敏感场景，设计系统化优化策略。核心原则：**先消峰（减少不必要的 IO/网络/解码），再并行（Isolate/并发），最后缓存（多级 LRU）。**

## 3. 首页缩略图网格（10000+ 资源）

### 3.1 虚拟滚动

**方案**：内置 `GridView.builder` + `SliverGridDelegateWithMaxCrossAxisExtent`，**无第三方包**。

`GridView.builder` 原生虚拟化：仅构建可视区 + `cacheExtent` 内的 Widget，滚出即销毁。网格几何完全确定性（`childAspectRatio` 固定），O(1) 跳转到任意偏移。

```dart
GridView.builder(
  key: const PageStorageKey('thumbnails'),
  controller: _controller,
  cacheExtent: MediaQuery.of(context).size.height * 1.5,
  gridDelegate: const SliverGridDelegateWithMaxCrossAxisExtent(
    maxCrossAxisExtent: 180,
    childAspectRatio: 2 / 3,
    mainAxisSpacing: 8,
    crossAxisSpacing: 8,
  ),
  itemCount: items.length,
  findChildIndexCallback: (key) {
    final id = (key as ValueKey<String>).value;
    final index = items.indexWhere((e) => e.id == id);
    return index == -1 ? null : index;
  },
  itemBuilder: (context, index) => KeyedSubtree(
    key: ValueKey(items[index].id),
    child: ResourceGridItem(item: items[index]),
  ),
)
```

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `maxCrossAxisExtent` | 180 | 自动 3~6 列，窗口 resize 自适应 |
| `childAspectRatio` | 2:3 | [@prd/01](../prd/01-资源库首页.md) §4.1.2 竖长方形 |
| `cacheExtent` | `MediaQuery.height * 1.5` | 动态适应分辨率，4K 屏预构建更多行防快滚闪白 |
| `PageStorageKey` | `'thumbnails'` | Tab 切回恢复滚动位置 |
| `ValueKey` | item.id | 增删时稳定身份 |

### 3.2 缩略图解码

缩略图按显示尺寸解码，避免全分辨率：

```dart
Image.file(
  path,
  cacheWidth: 180,
  cacheHeight: 270,
  fit: BoxFit.cover,
)
```

单张缩略图从全分辨率~数 MB 降至~97KB，10000 张内存压力从 GB 级降到 MB 级。

### 3.3 ImageCache 调优

```dart
PaintingBinding.instance.imageCache
  ..maximumSize = 200        // 条目数（降低防数字溢出）
  ..maximumSizeBytes = 300 * 1024 * 1024;  // 300MB
```

## 4. 查看器翻页性能

### 4.1 预加载窗口（对应 [@prd/04](../prd/04-资源查看器.md) §4.1.5）

维护滑动窗口 `[index-1, index, index+1, index+2]`，始终持 5 页生命周期：

- 窗口内每页 `ImageProvider.resolve()` + 持有 `ImageStreamListener`，pin 为 live 防 ImageCache LRU 淘汰
- 索引离开窗口时移除监听 + 取消进行中加载
- **并发解码信号量**（上限 2）：预加载不撑爆解码队列
- **快速滑动防抖**（200ms）：跳过中间帧，仅预取最终目标

### 4.2 自定义 PageImageProvider（关键设计）

`MemoryImage` 按 `Uint8List` 实例身份做键，同页面重加载生成新 `Uint8List` → 新键 → 缓存失效。必须自建稳定键：

```dart
class PageImageProvider extends ImageProvider<PageKey> {
  final String resourceId;
  final int pageIndex;
  final int? maxSide;

  @override
  Future<PageKey> obtainKey(ImageConfiguration configuration) =>
      SynchronousFuture(PageKey(resourceId, pageIndex));

  @override
  ImageStreamCompleter loadImage(PageKey key, ImageDecoderCallback decode) {
    return MultiFrameImageStreamCompleter(
      codec: _loadAsync(key, decode),
      scale: 1.0,
    );
  }

  Future<Codec> _loadAsync(PageKey key, ImageDecoderCallback decode) async {
    final bytes = await _contentProvider.loadPage(key.pageIndex);
    return decode(ImmutableBuffer.fromUint8List(bytes),
        targetWidth: maxSide, allowUpscaling: false);
  }
}

@immutable
class PageKey {
  final String resourceId;
  final int pageIndex;
  // == / hashCode 基于 resourceId + pageIndex
}
```

收益：缓存命中可靠、`evict(PageKey(...))` 精确清理、按显示尺寸解码。

### 4.3 extended_image 集成

```dart
ExtendedImageGesturePageView.builder(

  controller: _pageController,

  itemBuilder: (_, index) => ExtendedImage(

```

| 特性 | 映射 PRD |
|------|---------|
| `imageCacheName: 'viewer'` | 查看器独立缓存，退出 `clearMemoryImageCache('viewer')` 精确清空 |
| `ExtendedImageGesturePageView` | 翻页+缩放保持（[@prd/04](../prd/04-资源查看器.md) §4.1.2） |
| `enableLoadState` + `loadStateChanged` | 加载失败占位+重试（[@prd/04](../prd/04-资源查看器.md) §4.1.5） |
| `clearGestureDetailsCache()` | 退出时清理手势状态 |

### 4.4 超大图片（>10000px）（对应 [@prd/04](../prd/04-资源查看器.md) §5 第 2 条）

```
1. ImageDescriptor 探测尺寸（不解码）→ >8000px 进入超大路径
2. 始终 ResizeImage 按 viewport × dpr × zoom 解码
3. Isolate 用 image 包生成"查看母版"（如 4000px）缓存磁盘
4. 深度缩放分块 P2
```

### 4.5 退出清理（对应 [@prd/04](../prd/04-资源查看器.md) §5 第 7 条）

`ContentProvider.dispose()` 执行：
1. 移除预加载窗口全部 ImageStreamListener（unpin live）
2. 取消进行中加载
3. `clearMemoryImageCache('viewer')`
4. `clearGestureDetailsCache()`
5. 关闭 PDF 文档句柄 / archive 句柄 / SMB 流

## 5. 数据库性能（drift）

### 5.1 索引

| 表 | 索引 | 原因 |
|----|------|------|
| `ResourceTag` | `@TableIndex.sql CREATE INDEX ... ON resource_tags (tag_id, resource_id)` | 标签筛选的核心路径，默认 `(resource_id, tag_id)` unique 无法服务 `WHERE tag_id=?` |
| `Resource` | `@TableIndex(columns: {#sourceId})` | FK lookup + 级联删除 |
| `Resource` | `@TableIndex(columns: {#sourceId, #relativePath})` unique | 扫描去重（upsert 冲突检测） |
| `Resource` | `@TableIndex(columns: {#createdAt, #id})` | 按时间排序 + 键集分页 |
| `Resource` | `@TableIndex(columns: {#name, #id})` | 按名称排序 + 键集分页 |

### 5.2 标签交集查询

用 `GROUP BY HAVING COUNT(*) = N` 替代 [@prd/05](../prd/05-数据模型.md) §4.3 中 INTERSECT 示例：

```dart
Selectable<Resource> filterByTags(List<String> tagIds) {
  if (tagIds.isEmpty) return select(resources);

  final matchedIds = selectOnly(resourceTags)
    ..addColumns([resourceTags.resourceId])
    ..where(resourceTags.tagId.isIn(tagIds))
    ..groupBy([resourceTags.resourceId])
    ..having(countAll().equals(tagIds.length));

  return select(resources)
    ..where((r) => r.id.isInQuery(matchedIds))
    ..orderBy([(t) => OrderingTerm.desc(t.createdAt)]);
}
```

原因：
- INTERSECT 无法对组合结果添加 `ORDER BY/LIMIT`（SQL 规范限制），分页不友好
- GROUP BY 单次覆盖索引扫描（`tag_id` 索引），可自由排序分页
- "收藏 + 自定义标签"统一处理（收藏是普通 tagId）

### 5.3 键集分页（代替 offset）

10000+ 资源场景 offset 分页在深页退化（SQLite 扫描并丢弃前 N 行）。键集分页常数时间：

```dart
Future<List<Resource>> pageAfter({
  required String? lastCreatedAt,
  required String? lastId,
  required int pageSize,
}) {
  final q = select(resources)
    ..orderBy([
      (t) => OrderingTerm.desc(t.createdAt),
      (t) => OrderingTerm.asc(t.id),
    ])
    ..limit(pageSize);
  if (lastCreatedAt != null && lastId != null) {
    q.where((r) =>
      r.createdAt.isSmallerThanValue(DateTime.parse(lastCreatedAt!)) |
      (r.createdAt.equals(DateTime.parse(lastCreatedAt!)) &
       r.id.isBiggerThanValue(lastId!)));
  }
  return q.get();
}
```

需要 `(createdAt, id)` 复合索引。

### 5.4 Stream 策略

| 做法 | 是否推荐 |
|------|---------|
| `select(resources).watch()` 监听全量 10000 行 | ❌ 每次 Tag 关联变更重查全表 |
| `select(...).watchSingle()` 监听轻量 count（"12/56"徽标） | ✅ 每次返回一个整数，微秒级 |
| 当前可视页 `get()`（Future），滚动时取下一页 | ✅ |
| 可视页 `.watch()` 50 行实时监听 | ✅ 可选 |

### 5.5 扫描写入

```dart
Future<void> batchInsert(List<ResourcesCompanion> rows) async {
  await batch((b) => b.insertAll(resources, rows, mode: InsertMode.insertOrIgnore));
}
```

- 单 `transaction` + `batch.insertAll` 原子插入，Stream 合并为一次通知
- `insertOnConflictUpdate` 实现 upsert（对应 [@prd/05](../prd/05-数据模型.md) §4.4 同步：新增→创建，修改→更新，消失→isAvailable=false）
- 每 500~1000 行分块提交，进度流逐步更新（对应 [@prd/01](../prd/01-资源库首页.md) §4.4.2）
- `computeWithDatabase` 在 Isolate 共享同一逻辑库，写入对 UI 可见

### 5.6 数据库初始化配置

```dart
QueryExecutor _openConnection() {
  return NativeDatabase.createInBackground(
    _dbFile(),
    setup: (db) {
      db.execute('PRAGMA journal_mode = WAL;');
      db.execute('PRAGMA foreign_keys = ON;');
      db.execute('PRAGMA synchronous = NORMAL;');
    },
    readPool: 3,
  );
}
```

| PRAGMA | 作用 |
|--------|------|
| `journal_mode = WAL` | 并发读 + 单写分离；`readPool` 必需 |
| `foreign_keys = ON` | 级联删除生效，SQLite 默认关闭 |
| `synchronous = NORMAL` | WAL 下安全且更快 |
| `readPool: 3` | 大扫描 + 小查询并行不阻塞 |

## 6. 文件浏览缓存

### 6.1 核心思路

SMB `listDirectory` 单次往返即返回完整元数据（name/mtime/size/isDir 全部内联），优化目标是**消除重复往返**。最大收益场景：返回上级目录、左右双栏重复请求、面包屑快速点击。

采用**精简方案**：TTL 内存缓存 + 请求去重 + 导航防抖。**零新依赖，~100 行代码。**

### 6.2 TTL 内存缓存

```dart
class DirectoryCache {
  final Duration ttl;
  final _cache = <String, _Entry>{};

  DirectoryCache({this.ttl = const Duration(minutes: 2)});

  List<FileEntry>? get(String key) {
    final e = _cache[key];
    if (e == null) return null;
    if (DateTime.now().difference(e.cachedAt) > ttl) {
      _cache.remove(key);
      return null;
    }
    return e.data;
  }

  void put(String key, List<FileEntry> data) =>
      _cache[key] = _Entry(data, DateTime.now());

  void invalidate(String key) => _cache.remove(key);
  void clear() => _cache.clear();
}
```

**TTL 按源类型**：

| 源类型 | TTL | 理由 |
|--------|-----|------|
| Local（dart:io） | 30s | 本地 IO 近乎免费 |
| SMB | 2min | 网络昂贵，适度缓存 |
| FTP/WebDAV（未来） | 2min | 同 SMB |

### 6.3 请求去重

```dart
final _inflight = <String, Future<List<FileEntry>>>{};

Future<List<FileEntry>> _dedup(String key,
    Future<List<FileEntry>> Function() loader) {
  if (_inflight.containsKey(key)) return _inflight[key]!;
  final f = loader().whenComplete(() => _inflight.remove(key));
  _inflight[key] = f;
  return f;
}
```

两个 Widget（左侧树 + 右侧内容区）同时请求同一路径 → 共享一个 Future，仅一次 SMB 往返。

### 6.4 导航防抖

面包屑快速点击只触发最终目的地：

```dart
Timer? _debounce;
void navigate(String path) {
  _debounce?.cancel();
  _debounce = Timer(const Duration(milliseconds: 200), () {
    _doNavigate(path);
  });
}
```

### 6.5 一层调用双栏复用

`listDirectory` 一次返回全部数据。左侧目录树只展示文件夹（从完整缓存中 `where isDirectory` 过滤），右侧内容区展示完整列表。无额外 SMB 开销。

### 6.6 失效策略

| 触发 | 行为 |
|------|------|
| 源同步 / 手动刷新 | 清该源全部缓存 |
| 源禁用 / 删除 | 清缓存 + disconnect SMB 连接 |
| TTL 过期 | 单 key 自动过期 |
| Tab 切回 | 当前目录手动刷新一次 |

### 6.7 骨架屏

冷 miss 首次加载时显示骨架占位，避免白屏闪烁。SMB 不可达时置灰 + 警告图标，保留最后缓存内容只读展示。

### 6.8 性能收益

| 场景 | 优化前 | 优化后 |
|------|--------|--------|
| 进入子目录再返回 | 2× listDirectory | 1×（返回命中缓存） |
| 宽屏左右双栏 | 2× | 1×（去重共享） |
| 面包屑跳 3 层 | 3× | 1×（防抖跳中间） |
| Tab 切回数据源 | re-list | 缓存命中 |
| 同级目录间切换 | 每次 re-list | TTL 内命中 |

### 6.9 后续扩展（P2）

当前精简方案覆盖 80% 场景。以下扩展按需引入：

- **磁盘持久化**：`stash` + `stash_sqlite` 实现重启即用（复用 drift SQLite 栈）
- **Stale-While-Revalidate**：缓存命中时立即返回旧值 + 后台静默刷新（双 TTL：freshTtl 2min / staleTtl 30min）
- **mtime 快速校验**：`stat(dir).modified` 作为廉价负向信号，判断是否需要 relist
- **预测性预取**：兄弟目录预取、子节点一层预取、源连接时根目录 warm-up

## 7. 缩略图磁盘缓存（LRU）

> 对应 [@prd/08](../prd/08-设置页面.md) §4.1 · [@prd/01](../prd/01-资源库首页.md) §4.2

### 7.1 策略

- **淘汰策略**：LRU（最近最少使用）
- **默认容量**：500MB
- **可调档位**：500M / 1000M / 1500M / 2000M / 自定义
- **最小容量**：500MB，拒绝更小值
- **手动清理**：二次确认弹窗 → 清空所有缩略图文件

### 7.2 实现

`ThumbnailCacheService` 管理磁盘储存。可用 `stash` + `stash_file`（`LruEvictionPolicy` + `maxSize`），或手写 ~150 行文件级 LRU 避免新依赖。P0 阶段推荐手写精简版：

- 存储位置：`path_provider` 获取应用缓存目录
- 每个缩略图文件命名：`thumb_{resourceId}.jpg`
- 维护一个 JSON 索引文件记录访问时间
- 写入时检查总大小，超限时按 LRU 删除最后访问最旧的文件

### 7.3 内存层

缩略图缓存写入磁盘后，可见网格行的缩略图由 Flutter `ImageCache` 管理（已按显示尺寸解码，~97KB/张）。不另设独立内存缓存层。

## 8. 扫描入库性能

### 8.1 Isolate 后台执行

```
ResourcePicker 确认
  → computeWithDatabase 在 Isolate：
     ① 遍历文件（dart:io 或 dart_smb2）
     ② 过滤已入库项（sourceId + relativePath 去重）
     ③ 每 batch 500~1000 行 → transaction 写入
     ④ 每 batch 通过 Stream 推送进度
     ⑤ ThumbnailGenerator 后台生成缩略图
  → UI 监听进度 Stream，实时更新网格和"已扫描 N/M"计数
```

### 8.2 增量同步（对应 [@prd/05](../prd/05-数据模型.md) §4.4）

遍历源 rootPath → 对比已有 Resource 的 relativePath：
- **新增** → 创建 Resource
- **消失** → `isAvailable = false`（不删除，保留标签关联）
- **修改**（mtime/size 变化） → 更新 fileSize/fileCount + 重新生成缩略图
- **不变** → 跳过

## 9. 内存管理

### 9.1 多层策略

| 层 | 内容 | 工具 | 上限 |
|----|------|------|------|
| Flutter ImageCache | 解码后位图 | 内置，查看器用 `extended_image` named cache 隔离 | 300MB（全局）/ viewer 独立 |
| 查看器预加载字节 | 窗口内 Uint8List | LinkedHashMap LRU | ~50MB（5 页 × 10MB） |
| 缩略图磁盘 | jpg 文件 | ThumbnailCacheService | 500MB 默认 |
| 文件列表缓存 | List<FileEntry> | DirectoryCache | ~10MB（TTL 自淘汰） |

### 9.2 生命周期

| 事件 | 响应 |
|------|------|
| `ContentProvider.dispose()` | 移除预加载监听、取消加载、`clearMemoryImageCache('viewer')`、关闭 PDF/SMB/ZIP 句柄 |
| `WidgetsBindingObserver.didHaveMemoryPressure` | `imageCache.clear()` + 清 DirectoryCache |
| `AppLifecycleState.paused` | `imageCache.clear()` 释放内存，恢复时廉价重解码 |
| 源删除 | 清 DirectoryCache + 清该源缩略图 + disconnect SMB |

### 9.3 超大图片 OOM 防护

- 始终 `ResizeImage` + `cacheWidth/cacheHeight` 按显示尺寸解码
- `ImageDescriptor` 探测尺寸后分流：≤8000px 直接解码，>8000px Isolate 降质
- PDF 通过 `pdfrx` 按请求尺寸渲染，天然不过大

## 10. 架构落点总结

| 组件 | 性能职责 |
|------|---------|
| `ThumbnailCacheService` | 磁盘 LRU，容量管理，手动清理 |
| `ThumbnailRepository` | 委托 CacheService，Isolate 生成 |
| `ContentProvider` | loadPage 按需，dispose 全释放 |
| `PageImageProvider`（`shared/content_provider/`） | 稳定键 ImageProvider，按尺寸解码 |
| `DatabaseService` | PRAGMA 配置，后台 Isolate，readPool |
| `TagRepository` | GROUP BY HAVING COUNT 交集查询 |
| `FilesystemRepository` | DirectoryCache TTL + 去重 + 防抖 |
| `FileSourceFactory` | SMB 连接池复用 |
| `ScanResourcesUseCase` | Isolate + 分块提交 + 进度流 |
| `ViewerViewModel` | 预加载窗口 + 滑动 debounce |
