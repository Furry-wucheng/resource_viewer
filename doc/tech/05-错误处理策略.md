# 05 — 错误处理策略

> 日期：2026-06-19 · 状态：已定稿 · 依据：[@02-架构设计.md](02-架构设计.md) · 覆盖 [@prd/01](../prd/01-资源库首页.md)~[@prd/08](../prd/08-设置页面.md) 全部错误场景

---

## 1. 概述

定义从 Service → Repository → UseCase → ViewModel → View 的**统一错误传播约定**，确保各层错误处理一致、可追溯、可降级。

核心原则：**预期错误不走异常栈（用 Result），编程错误走异常（用 throw）。**

---

## 2. 错误分类

### 2.1 来源域

| 域 | 来源 | 典型场景 |
|----|------|---------|
| I/O | 文件系统、SMB 网络 | 路径不存在、断网、权限不足、超时 |
| 数据 | drift 数据库 | 约束冲突、迁移失败、磁盘满 |
| 媒体 | 图片解码、PDF 渲染、压缩包 | 损坏文件、不支持的格式、加密 PDF |
| 用户输入 | 表单校验 | 标签名空/重复/与内置冲突、缓存容量 < 500MB |
| 状态 | 业务状态 | 资源不可达 (isAvailable=false)、空文件夹 |

### 2.2 严重级别与处理策略

| 级别 | 定义 | 策略 | 示例 |
|------|------|------|------|
| **Fatal** | 数据库损坏、迁移失败 | Abort：阻塞全屏 | 数据库打开失败 |
| **Recoverable** | SMB 超时、图片加载失败 | Degrade / Retry | 降级 UI + 用户重试 |
| **Expected** | 标签名重复、空文件夹 | Skip / Toast | 不打断流程 |

## 3. DomainError 类型树

```dart
sealed class DomainError {
  const DomainError(this.message, {this.cause});
  final String message;
  final Object? cause;
}

/// SMB / 本地文件系统类
final class SourceUnreachableError extends DomainError {
  const SourceUnreachableError(super.message, {super.cause});
}
final class SourceAuthError extends DomainError {
  const SourceAuthError(super.message, {super.cause});
}
final class FileNotFoundError extends DomainError {
  const FileNotFoundError(super.message, {super.cause});
}
final class FileAccessDeniedError extends DomainError {
  const FileAccessDeniedError(super.message, {super.cause});
}
final class NetworkTimeoutError extends DomainError {
  const NetworkTimeoutError(super.message, {super.cause});
}

/// 媒体加载类
final class MediaLoadError extends DomainError {
  const MediaLoadError(super.message, {super.cause, required this.mediaType});
  final MediaType mediaType; // image / pdf / archive / video
}
final class MediaEncryptedError extends DomainError {
  const MediaEncryptedError(super.message, {super.cause, required this.mediaType});
  final MediaType mediaType; // 区分 PDF/ZIP 加密，给出针对性提示
}

/// 校验类
final class ValidationError extends DomainError {
  const ValidationError(super.message);
}

/// 用户取消操作（扫描中取消、文件浏览器返回）
final class OperationCancelledError extends DomainError {
  const OperationCancelledError([super.message = '操作已取消']);
}

/// 磁盘空间不足（缩略图缓存无法写入）
final class InsufficientStorageError extends DomainError {
  const InsufficientStorageError(super.message, {super.cause});
}

/// 不支持的格式（7z/RAR 暂不支持）
final class UnsupportedFormatError extends DomainError {
  const UnsupportedFormatError(super.message, {super.cause, required this.format});
  final String format; // '7z' / 'rar' 等
}

/// 数据库类（Fatal）
final class DatabaseError extends DomainError {
  const DatabaseError(super.message, {super.cause});
}
```

`MediaLoadError` 不按图片/PDF/Archive/视频分裂子类型（四者在 ViewModel 中处理路径一致），用 `mediaType` 字段（独立的 `MediaType` 枚举，与 `ResourceType` 解耦）区分提示文案。

```dart
enum MediaType { image, pdf, archive, video }
```

## 4. Result\<T\> 定义

```dart
sealed class Result<T> {
  const Result();
}

class Ok<T> extends Result<T> {
  const Ok(this.value);
  final T value;
}

class Err<T> extends Result<T> {
  const Err(this.error);
  final DomainError error;
}
```

手写，零依赖。`sealed class` 确保 `switch` 穷举编译期检查。

## 5. 各层约定

### 5.1 Service 层：只抛出，不捕获

```dart
class SmbFileService {
  // 直接抛 Smb2Exception，不做域转换
  Future<List<Smb2DirEntry>> listDirectory(String path) async {
    return _pool.listDirectory(path);
  }
}
```

Service 是无状态函数，不引入 Domain 概念。原始异常向上透传。

### 5.2 Repository 层：捕获 + 转为 `Result<T>`

```dart
class FilesystemRepository {
  final FileSourceFactory _factory;

  Future<Result<List<FileEntry>>> listDirectory(
      String sourceId, String path) async {
    try {
      final source = _factory.create(sourceId);
      final entries = await source.listDirectory(path);
      return Ok(entries);
    } on Smb2Exception catch (e) {
      if (e.type == Smb2ErrorType.auth) {
        return Err(SourceAuthError('SMB 认证失败', cause: e));
      }
      if (e.isConnectionError) {
        return Err(SourceUnreachableError('SMB 源不可达', cause: e));
      }
      return Err(DomainError('SMB 错误: ${e.message}', cause: e));
    } on FileSystemException catch (e) {
      if (e.osError?.errorCode == 2 || e.osError?.errorCode == 3) {
        return Err(FileNotFoundError('路径不存在: $path', cause: e));
      }
      return Err(FileAccessDeniedError('权限不足: $path', cause: e));
    } catch (e) {
      return Err(DomainError('未知错误: $e', cause: e));
    }
  }
}
```

**规则**：
- 必须用 `Result` 包装返回，不抛 — ViewModel 不能 `try/catch`
- `cause` 保留原始异常，便于日志追踪
- 未知异常兜底为 `DomainError` 泛型实例

### 5.3 ViewModel 层：`Result` → `UiState`

```dart
enum UiState { idle, loading, success, error }

abstract class BaseViewModel extends ChangeNotifier {
  UiState _state = UiState.idle;
  UiState get state => _state;

  String? _errorMessage;
  String? get errorMessage => _errorMessage;

  DomainError? _lastError;
  DomainError? get lastError => _lastError;

  bool get canRetry => switch (_lastError) {
    SourceUnreachableError() || FileNotFoundError()
    || NetworkTimeoutError() || MediaLoadError() => true,
    _ => false,
  };

  void _startLoading() {
    _state = UiState.loading;
    _errorMessage = null;
    notifyListeners();
  }

  void _setResult<T>(Result<T> result) {
    switch (result) {
      case Ok():
        _state = UiState.success;
        _errorMessage = null;
      case Err(:final error):
        _state = UiState.error;
        _errorMessage = _mapError(error);
        _lastError = error;
    }
    notifyListeners();
  }
}
```

子类 ViewModel 继承复用：

```dart
class FileBrowserViewModel extends BaseViewModel {
  Future<void> openDirectory(String path) async {
    _startLoading();
    final result = await _repo.listDirectory(_sourceId, path);
    _setResult(result);
  }
}
```

**`_mapError` 文案映射**（对应 PRD 各提示）：

```dart
String _mapError(DomainError err) => switch (err) {
  SourceUnreachableError() =>
      '数据源不可达，请检查网络连接',
  SourceAuthError() =>
      'SMB 认证失败，请检查用户名和密码',
  FileNotFoundError() =>
      '路径不存在或已被移除',
  FileAccessDeniedError() =>
      '权限不足，无法访问该路径',
  NetworkTimeoutError() =>
      '连接超时，请稍后重试',
  MediaLoadError(mediaType: MediaType.pdf) =>
      'PDF 加载失败',
  MediaLoadError(mediaType: MediaType.archive) =>
      '压缩包读取失败',
  MediaLoadError(mediaType: MediaType.video) =>
      '视频加载失败',
  MediaLoadError() =>
      '图片加载失败',
  MediaEncryptedError(mediaType: MediaType.pdf) =>
      '此 PDF 已加密，暂不支持查看',
  MediaEncryptedError(mediaType: MediaType.archive) =>
      '此压缩包已加密，暂不支持查看',
  MediaEncryptedError() =>
      '此文件已加密，暂不支持查看',
  DatabaseError() =>
      '数据库异常，请重启应用',
  _ => '未知错误',
};
```

文案暂硬编码中文，未来国际化集中在此函数替换。

### 5.4 View 层：按状态渲染

```dart
Widget _buildBody(BuildContext context, FileBrowserViewModel vm) {
  return switch (vm.state) {
    UiState.idle => const SizedBox.shrink(),
    UiState.loading => const FileBrowserSkeleton(),
    UiState.error => _ErrorView(
        message: vm.errorMessage!,
        canRetry: vm.canRetry,
        onRetry: () => vm.retry(),
      ),
    UiState.success => _buildContent(vm),
  };
}
```

`_ErrorView` 根据 `lastError` 类型调整视觉：

| lastError 类型 | 视觉 |
|---------------|------|
| `SourceUnreachableError` | 置灰 + 警告图标 + 保留最后缓存内容只读展示 |
| `MediaLoadError` | 占位图 + 重试按钮（该页不自动跳过） |
| `ValidationError` | Toast（不进入 error 状态） |
| 其他 Recoverable | 居中错误提示 + 重试按钮 |
| `DatabaseError` | 全屏阻塞"数据库损坏" |

## 6. 特殊场景

### 6.1 源不可达（来源 [@prd/01 §4.4.3](../prd/01-资源库首页.md) · [@prd/03 §5 第 1 条](../prd/03-数据源管理.md)）

```
SMB listDirectory 抛 Smb2Exception
  → Repository 返回 Err(SourceUnreachableError)
  → ViewModel 中额外：SourceRepository.markUnavailable(sourceId)
  → 该源所有 Resource.isAvailable = false
  → 首页：该源资源置灰 + 标记
  → 文件浏览器：显示断网占位
  → 用户手动同步或网络恢复 → markAvailable
```

### 6.2 图片/PDF/视频加载失败（来源 [@prd/04 §4.1.5](../prd/04-资源查看器.md)）

> ContentProvider 和 media_kit Player 是 `shared/` 层抽象，不在 Repository 边界内，其异常无法由 Repository 包装为 `Result`。因此 ViewerViewModel 直接 try/catch 是**允许的例外**（见 §10 #1）。

```
ContentProvider.loadPage(index) 或 media_kit Player 抛异常
  → ViewerViewModel 捕获 → pageErrors[index] = MediaLoadError
  → 未翻页：当前页显示"加载失败"占位 + 重试按钮
  → 用户翻到其他页：正常加载，不阻断
  → 用户点重试 → reloadPage(index)
```

### 6.3 扫描入库（来源 [@prd/03 §4.5](../prd/03-数据源管理.md)）

```
Isolate 遍历文件
  → 每项：try 处理 → 成功入 successList / 失败入 failureList
  → 返回 ScanResult(successCount, skipCount, failures)
  → UI: toast "成功导入 8 个资源，2 个跳过，1 个失败"
  → failures 中的项不创建 Resource，仅记录日志
```

```dart
class ScanResult {
  final int successCount;
  final int skipCount;   // 空文件夹/已入库
  final List<({String path, DomainError error})> failures;
}
```

### 6.4 标签名校验（来源 [@prd/02 §5 第 1~8 条](../prd/02-标签系统.md)）

**ViewModel 层操作前校验**，不到 Repository：

```dart
String? validateTagName(String name, List<String> existingNames) {
  if (name.trim().isEmpty) return '标签名不能为空';
  if (name == '收藏') return "'收藏'是系统内置标签，请换一个名称";
  if (existingNames.contains(name)) return "标签'$name'已存在";
  if (name.length > 20) return '标签名不能超过 20 个字符';
  return null; // 通过
}
```

校验失败 → `ValidationError` → View `SnackBar` / 输入框下方红字，不进入 `UiState.error`。

### 6.5 Fatal（数据库损坏）

```
AppDatabase 初始化失败
  → runApp 前捕获
  → 显示全屏 FatalErrorPage("数据库损坏，请重新安装应用")
  → 无重试按钮
```

```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  try {
    final db = AppDatabase();
    // drift 构造是 lazy 的，需显式预热连接以触发实际打开/迁移，
    // 否则初始化错误会在首次查询时抛出（已在 runApp 内部，无法兜住）
    await db.customSelect('SELECT 1').get();
    runApp(ResourceViewerApp(db: db));
  } catch (e) {
    runApp(FatalErrorApp(error: e));
  }
}
```

## 7. 流式操作（Stream 进度）

扫描等长时操作通过 `Progress<T>` 替代 `Result<T>`：

```dart
sealed class Progress<T> {
  const Progress();
}
class ProgressUpdate<T> extends Progress<T> {
  const ProgressUpdate(this.value, this.current, this.total);
  final T value;
  final int current;
  final int total;
}
class ProgressDone<T> extends Progress<T> {
  const ProgressDone(this.result);
  final T result;
}
class ProgressError extends Progress<Never> {
  const ProgressError(this.error);
  final DomainError error;
}
```

```dart
Stream<Progress<ScanResult>> scanResources(String sourceId) async* {
  for (var i = 0; i < entries.length; i++) {
    try {
      final resource = await _processOne(entries[i]);
      yield ProgressUpdate(resource, i + 1, entries.length);
    } on Exception catch (e) {
      // 单项失败不中止流，收集到最终 ScanResult
      _failures.add((path: entries[i].path, error: _toDomainError(e)));
    }
  }
  yield ProgressDone(ScanResult(...));
}
```

| 级别 | 工具 | 内容 |

| Debug | `debugPrint` | Service 异常详情 |

## 10. 注意项

1. **ViewModel 不 try/catch** — 所有预期错误走 `Result`，Exception 只在 Service 层和 Repository 内部。**唯一例外**：`ContentProvider.loadPage()` 和 media_kit `Player` 的异常由 ViewerViewModel 直接 try/catch（因为 ContentProvider 是 `shared/` 层抽象，不在 Repository 边界内，无法由 Repository 包装为 `Result`）。详见 §6.2。
2. **`cause` 始终保留** — 调试时靠原始异常定位根因
3. **`canRetry` 由 `_lastError` 类型决定** — 校验错误不提供重试
4. **流式操作失败不中止** — 单个失败记录到 `failures`，整体继续
5. **Fatal 必须在 `runApp` 前拦截** — 详见 §6.5，`main()` 需 `async` + `await db.ensureOpen()` 预热
6. **Toast / 行内红字不走 `UiState.error`** — 校验类错误不是页面级错误
